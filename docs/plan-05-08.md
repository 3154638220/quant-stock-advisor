## 3. W1：因子库扩展（Alpha158 基准对齐）

### 3.1 背景与差距评估

当前因子库约 80 个特征，覆盖价量动量/基本面/资金流/概念 breadth 四族。参考 Microsoft Qlib Alpha158 和国内主流因子体系，当前缺失的系统性因子族包括：

| 缺失因子族 | 代表因子 | A 股文献支持 |
|-----------|---------|------------|
| **质量因子** | ROE 稳定性、Accruals、毛利率变化、资产增长率 | 强（Fama-French 5F A 股复现均显著） |
| **盈利修正/惊喜** | 业绩预告上下调幅度、分析师超预期 | 极强（A 股分析师跟踪月度延续性显著） |
| **短期反转** | 上月收益（做空信号）、周收益反转 | A 股短期反转因子 IC 高于美股 |
| **成交量异常** | 量价背离、换手异常 z-score、历史量比 | A 股散户拥挤行为明显 |
| **价格位置/区间** | 52 周高低位置、布林带位置 | 已有部分实现，需系统化 |
| **流动性风险** | Amihud 非流动性、买卖价差代理 | 小盘股溢价来源之一 |
| **分析师覆盖** | 机构覆盖数变化、目标价上调比例 | 若数据可得，高 Alpha 因子 |

### 3.2 实施计划

```text
Phase 1（1 周）：质量因子 & 盈利动量
  src/features/quality_factors.py
  - roe_stability（ROE 过去 8 季度标准差的负值）
  - accruals（应计项目比率 = (净利润 - 经营现金流) / 总资产）
  - gross_margin_change（毛利率月度/季度变化）
  - asset_growth_rate（总资产同比增速，负向因子）
  - earnings_surprise（最新季报 EPS vs 一致预期/历史，可从 fundamental 表构建）

Phase 2（1 周）：短期反转 & 量能异常
  src/features/reversal_volume_factors.py
  - short_term_reversal_1m（上月超额收益，做空）
  - short_term_reversal_1w（上周超额收益，做空）
  - volume_spike_ratio（近 5 日成交额均值 / 近 60 日，高值=情绪过热）
  - turnover_anomaly（成交额相对自身历史分布的 z-score）
  - price_volume_divergence（价涨量缩/价跌量增）

Phase 3（1 周）：流动性 & 价格位置扩展
  src/features/liquidity_position_factors.py
  - amihud_illiquidity（日均 |收益| / 成交额，过去 20 日均值）
  - high52w_ratio（当前价 / 52 周高点）
  - low52w_ratio（当前价 / 52 周低点）
  - price_range_width（(52周高-低) / 52周均价，低值=低波动、稳定）
```

### 3.3 验收 Gate

每个新因子族必须通过：

3. 候选池覆盖率 > 60%
4. M5 增量 delta 测试（Rank IC delta vs 当前 hard-cap baseline 不显著为负）

## 4. W3：LightGBM 对比评估

### 4.1 背景

当前模型层核心为 ExtraTrees + ElasticNet。ExtraTrees 的已知缺点：
- 随机分裂导致特征利用效率低于 LightGBM
- 无原生 ranking loss 支持
- 训练速度慢，超参数网格搜索成本高

LightGBM 在国内主流量化机构（九坤/幻方/明汯）和 Qlib 基准测试中，在月频 tabular 选股任务上通常优于或持平 ExtraTrees，且训练速度快 5–20x。

### 4.2 实施方案

```python
# 在 src/models/baseline/ 新增 lightgbm_ranker.py
# 支持 rank:pairwise（LambdaRank）和 binary 分类两种 loss
# 使用与 M5 ExtraTrees 相同的 walk-forward 切分

models_to_compare = [
    "extratrees_excess",       # 当前最强
    "elasticnet_excess",       # 当前基准
    "lgbm_regression_excess",  # LightGBM 回归 excess
    "lgbm_rank_pairwise",      # LightGBM lambdarank
    "lgbm_binary_top20",       # LightGBM top-20% 分类
]
```

关键超参（walk-forward 内 CV 确定，不在测试月调参）：
- `num_leaves`：31/63/127
- `min_child_samples`：50/100/200（防过拟合关键）
- `learning_rate`：0.05/0.1
- `feature_fraction`：0.7/0.8/1.0

验收 gate：Rank IC delta vs ExtraTrees ≥ 0（统计显著 p < 0.1）

## 5. W4：结构化公告事件因子（M16 前置探索）

### 5.1 候选因子设计

```python
features_event = [
    # 业绩预告（PIT：以公告日为信号可用日）
    "earnings_guidance_direction",   # 上调/下调/持平，过去 60 天内最新公告

    "reduction_plan_flag",           # 过去 30 天有大股东减持计划（负向）

```

### 5.2 PIT 安全设计

```text
1. 以公告日（announce_date）而非事件生效日（effective_date）作为信号截止日
2. 信号日 t 只使用 announce_date <= t 的公告
3. 在 src/data_fetcher/migrations.py 增加事件表 DDL 与 PIT 时间戳字段
4. 实现 src/features/event_factors.py
5. 通过 M1 质量 gate（PIT 审计、覆盖率 > 40%、缺失标记正确）后方可接入 M5
```

## 6. W5：因子 IC 衰减 & 因子正交化治理

### 6.1 当前问题

当前 80 个特征通过 winsorize + z-score + 线性 blend 进入模型，存在两个未治理的问题：

1. **IC 衰减不均**：部分因子（如短期反转）在信号日 t+5/t+10 衰减极快，在月频下几乎无效；但这些因子仍占用特征预算，可能带来噪声
2. **因子冗余/多重共线性**：价量相关系数矩阵中，动量类因子两两相关可达 0.7+，导致模型权重分配不稳定

### 6.2 治理方案

```text
新增 scripts/run_factor_ic_audit.py：
  - 输出每个特征的 IC 时序、IC IR、IC 衰减曲线（lag 0/5/10/20 交易日）
  - 输出特征间 Spearman 相关矩阵（截面均值）
  - 标记 IC IR < 0.2 的弱因子（候选降级为缺失标记）
  - 标记两两相关 > 0.7 的冗余对（候选剔除一个）

可选：PCA 正交化降维保留 95% 方差，对比降维前后 Rank IC
```

## 7. W6：Oracle Gap 分解诊断

当前 Oracle 可分性上限（U1 Top20 月均超额 ~74%），而 hard-cap baseline 约 2.05%——即模型仅"使用"了 Oracle 上限的 ~2.8%。这个 gap 的瓶颈来源需要系统性诊断，以决定后续投入方向。

```text
新增 scripts/run_oracle_gap_analysis.py：

Level 1：因子层 gap
  - 对每个单因子计算条件 IC（P(top20 | top quartile 该因子)）
  - 找出"能预测 oracle top-20"最强的单因子
  - 找出与 oracle top-20 相关性最高的特征组合（稀疏回归）

Level 2：模型层 gap
  - 强制训练"oracle imitator"（以 oracle top-20 标签训练，不用 forward return）
  - 比较 oracle imitator vs 当前 M5 模型的预测相关性
  - 若 oracle imitator Rank IC 远高于 M5，说明模型层有改进空间
  - 若 oracle imitator Rank IC 也不高，说明因子层是瓶颈

Level 3：信息层 gap（定性）
  - oracle 选出的股票有什么共同特征？（事后归因）
  - 是否存在当前数据库没有的信息维度（如订单流、大宗数据、新闻）？
```

## 8. 对当前方法的技术评估

> 这一节评估当前项目在国内量化选股前沿中的定位，以及深度学习等方向是否值得引入。

### 8.1 当前方法相对前沿的优势

1. **规范的研究治理框架**：PIT-safe 数据管线、walk-forward 时序 CV、promoted registry、OOS 跟踪追踪——这套治理在散户和小机构量化中属于前 10% 水平，大多数公开研究缺乏这种严格性。

2. **符合月频 tabular 任务的模型选择**：在月频换仓的 A 股截面选股任务中，ExtraTrees/ElasticNet 是主流且有竞争力的选择。国内头部量化私募（九坤、幻方、明汯）在月频层面也以 GBM 类模型为主。

3. **成本模型合理**：16 bps 散户成本假设比很多学术研究更接近真实情形。

### 8.2 当前方法相对前沿的差距

| 维度 | 当前状态 | 前沿水平 | 差距性质 |
|------|---------|---------|---------|
| **因子库规模** | ~80 特征（4 族） | Qlib Alpha158（158 个）；头部私募 500–2000+ | 重要，可填补 |
| **因子挖掘方法** | 手工设计 | 遗传规划 / 强化学习因子搜索（AlphaGen、WorldQuant BRAIN） | 暂时不必要，见 8.3 |
| **模型排序损失** | 回归 excess（ElasticNet）/ 随机森林 | LambdaRank / ListNet / Pairwise NDCG + 行业约束 | 重要，可改进 |
| **市场微观结构因子** | 无分钟线 | 10/30/60 分钟 VWAP 因子、Level-2 订单不平衡 | 基础设施成本高 |
| **文本/事件 NLP 因子** | 无 | 公告 NLP、Xueqiu 情绪、研报超预期 | 高 Alpha 但开发成本高 |
| **图神经网络** | 无 | 供应链 GNN、行业关联传导 | 研究前沿，工程成本极高 |
| **因子组合框架** | 等权/简单 blend | Barra 多因子模型、Black-Litterman、因子动量动态权重 | 部分可改进 |

### 8.3 深度学习因子挖掘：当前是否值得引入？

**结论：月频任务下，当前阶段不应把深度学习作为第一主线，但有几个具体场景值得探索。**

理由：

1. **月频数据太稀疏**：64 个训练月 × ~3000 股 = ~19 万样本，深度模型在此量级极易过拟合。tabular 深度模型（TabNet）的 A 股实测表现通常不稳定超过 LightGBM。

2. **当前瓶颈是因子质量，不是模型复杂度**：Rank IC ≈ 0.11 的限制来自信息缺失（因子库覆盖不足），而不是 ExtraTrees 的容量上限。在信息没有增加的情况下，换更复杂的模型只会引入过拟合风险。

3. **M6 XGBoost rank NDCG 失败的教训**：高复杂度模型（XGBoost rank NDCG）已经证明在当前特征集下换手过高、无法稳定改善——深度 LTR 模型会面临同样问题。

**值得探索的 DL 具体场景**（待因子库扩展至 150+ 特征后）：

| 场景 | 方法 | 时机 |
|------|------|------|
| **日线序列特征提取** | 轻量 TCN（已有 `src/models/timeseries/lstm_tcn.py`）提取日内价量序列的非线性特征作为月频特征 | 当前 TCN 模块存在但未接入 M5，可验证 |
| **公告文本特征** | FinBERT / ERNIE-Finance 对业绩预告文本分类（超预期/低于预期/中性） | 待 M16 事件数据完善后 |
| **概念扩散图** | 轻量 GNN 在概念-股票二部图上传播特征 | 待 M13-B PIT-safe 复检通过后 |
| **因子挖掘自动化** | Symbolic regression（非深度学习）用 deap/gplearn 对当前价量数据做公式搜索 | W1 因子库扩展完成后，可作平行实验 |

### 8.4 参考前沿同类项目做法

**Microsoft Qlib（开源基准）**：
- 使用 Alpha158 因子库（158 个价量技术因子）作为基准特征集
- 主要模型：LightGBM、TabNet、LSTM（日频）、DoubleEnsemble
- 月频任务上，LightGBM 仍是最稳定的基线
- 与当前项目最大差距：因子库规模（158 vs ~80）和 ranking loss 使用

**国内头部量化私募做法（九坤/幻方/明汯基于公开信息推断）**：
- 月频选股主要依赖：价量技术因子 + 基本面因子 + 事件因子（业绩预告） + 行为金融因子（龙虎榜/北向）
- 机器学习层：主要是 GBM + 浅层 MLP，很少用深度序列模型在月频
- 研究治理：严格 PIT、walk-forward、大量 IC/ICIR 分析

**WorldQuant BRAIN 风格因子搜索**：
- 使用遗传规划在公式空间中搜索 Alpha，评估 IC 后存活
- 适合资源充足（多机并行）的机构
- 小资金散户替代方案：gplearn 库 + 单机遗传规划，可作为 W1 扩展后的探索

## 9. 模型路线（更新）

### 9.1 当前模型分工

| 层级 | 模型 | 当前结论 |
|------|------|---------|
| 稳定底座 | ExtraTrees regression excess（U1） | 当前生产基准，Rank IC ≈ 0.11 |
| 可解释底座 | ElasticNet regression excess（U1） | 更可解释，Rank IC 接近 |
| 待评估 | **LightGBM regression / lambdarank** | OOS 等待期评估目标 |
| 辅助特征 | LHB 因子（覆盖 12-27%，辅助权重） | 已保留，不独立 promote |
| 概念信号 | M13-B 概念因子（6 月后 PIT-safe 复检） | 有条件等待 |
| 冻结 | M6 XGBoost rank NDCG（LTR） | 系统性失败，不重启 |

### 9.2 下一版模型原则（更新）

1. M5 ExtraTrees + ElasticNet 保持稳定底座，不替换，只并列比较 LightGBM。
2. LightGBM lambdarank 作为"更强 LTR"替代 M6 XGBoost——但必须同样通过 U1 + top20 + walk-forward gate，换手 < 70%。
3. 任何模型层变化必须通过完整 M5 gate，不允许只看训练集表现。
4. 因子库扩展（W1）优先于模型替换，因为当前瓶颈是信息而非容量。
