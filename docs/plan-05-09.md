## 1. 核心背景：IC Gate ≠ 生产级 Baseline Gate

这是本计划最重要的认知澄清，必须在执行前明确。

| 对比维度 | W1 IC Gate（已完成） | 生产 Baseline Gate（本计划核心） |
|---------|---------------------|--------------------------------|
| **目的** | 验证单个因子信号质量 | 验证因子加入 M8 生产模型后整体收益提升 |
| **基准** | `price_volume_only`（8 个价量特征 + ExtraTrees） | `M8_regime_aware_fixed_policy__indcap3`（同等特征集） |
| **模型** | ExtraTrees 和 ElasticNet 原始输出 | M8 regime-aware policy + indcap3 行业分散 |
| **判断指标** | 单因子 IC IR、Rank IC delta vs PV-only | after-cost 月超额 delta vs 生产 baseline（2.05%/月） |
| **当前结论** | quality 通过（delta IC +0.0027, 69% pos）；Phase 2-3 Rank IC 正向但 after-cost 小幅为负 | **未执行，是本计划 T1** |

**关键含义**：M8 的月超额来源不是纯 Rank IC，而是 regime-aware 选股策略 + indcap3 集中度控制。W1 因子是否改善 M8 是独立的问题，必须跑 M8 管线才能知道。

### I1：IC Gate 规程增加「生产 Baseline Gate」步骤（优先级：🔴 高）

**问题**：当前 IC Gate（`src/features/ic_f1_gate.py`）只验证单因子 IC 质量，基准是 `price_volume_only`。但正如本计划第 1 节所示，通过 IC Gate 不等于改善生产 M8 模型。IC Gate 和 Baseline Gate 是两个不同的验收层次，当前只做了前者就标记"通过"容易产生误解。

**改进方案**：

在因子新增工作流中明确两阶段 gate：
- **Stage A（IC Gate）**：单因子信号质量，基准 price_volume_only + ExtraTrees，门槛 delta IC IR ≥ 0.3 且 coverage ≥ 60%。通过后因子进入「验证候选」。
- **Stage B（M8 Baseline Gate）**：因子加入 M8 生产管线后的整体收益验证，基准当前 promoted production config，门槛 after-cost delta ≥ 0。通过后因子进入「promotion 候选」。

在 `configs/experiments/README.md` 和因子新增 SOP 中补充此两阶段说明。

### I4：Oracle Gap Level 2 分析（模型层诊断）（优先级：🟡 中）✅ 已完成 2026-05-11

**问题**：W6 只完成了 Level 1（单因子条件 IC），发现"单因子 prob_lift 0.001–0.004，确认因子层为主要瓶颈"。但 Level 2（oracle imitator）仍缺失：

- Level 2 的价值：直接比较"用 oracle 标签训练的模型"的 Rank IC 与当前 M5 模型，可量化模型层 gap
- 若 oracle imitator Rank IC 也只有 ~0.11，说明当前特征集本身的信息上限就是 0.11，加再多特征也无法突破——是信息层瓶颈
- 若 oracle imitator Rank IC 达到 0.2+，说明现有特征理论上有更多信息可利用，是模型层/训练方法的瓶颈

**已实现**：在 `scripts/run_oracle_gap_analysis.py` 增加 `--level2` 选项，walk-forward ExtraTrees 框架。修复了两个问题：
1. NaN 处理：从 `dropna(subset=236 cols)` 改为中位数填充 + 60% 覆盖率过滤
2. 数据泄露：`label_forward_1m_*` 列被排除出特征集（原本导致 baseline reg IC=1.0）

**Level 2 结论（2026-05-11）**：

| 指标 | Oracle Imitator (Classifier) | Baseline Reg (Regressor) |
|------|------------------------------|--------------------------|
| Rank IC mean | **−0.075** | **+0.025** |
| IC IR | −0.48 | +0.18 |
| IC std | 0.155 | 0.140 |
| Test months | 27 | 27 |

**诊断：信息层瓶颈**。特征集本身的信息上限 ≈ 0.025 baseline IC，远低于 M5/M8 管线实际达到的 ~0.10 IC。这意味着：

1. **Oracle imitator 显式负向**（IC = −0.075）：直接用 oracle top-20 标签训练的分类器不仅没有正向预测力，反而负相关。原因：oracle top-20 标签极端不平衡（无条件概率 0.14%，每月仅 ~7 只正样本），205 个特征中的噪声淹没了微弱信号。
2. **Baseline ExtraTrees 仅 0.025 IC**：远低于 M5 管线的 ~0.10，说明 M8 regime-aware policy + indcap3 + ensemble 贡献了约 4x 的 IC 提升（0.025 → 0.10）。这是模型层价值的有力证据。
3. **信息层 vs 模型层**：特征原始信息上限 ~0.025，但 M8 管线通过 regime 择时、行业集中度控制、多模型集成将可用信息放大到 ~0.10。这确认了「信息层是主瓶颈，但模型层有显著放大作用」的判断。
