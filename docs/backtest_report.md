## 四、根本原因诊断（历史口径 vs 当前口径）

### 4.0 当前回测口径说明（重要）

- 本报告数值来自 `**scripts/run_backtest_eval.py` 改进版**：`composite_extended` 线性打分 + Top-50 等权 + **月末调仓** + **预过滤** + **换手约束** + `close_to_close`。
- **未使用** `sort_by = xgboost` 或 `tree_score` 排序；因此 **4.1 节**保留为「历史工件/并行系统」诊断，不等同于本次表格的策略链路。

### 4.1 XGBoost 模型 Rank IC 为负（历史记录）

查看 `data/models/xgboost_panel_ee2108f515be/bundle.json`（若工件仍存在）：

```json
"metrics": {
    "train_rank_ic": -0.16114580398302147,
    "val_rank_ic": -0.07358242336614988
}
```

**核心结论（适用于 xgboost 排序链路）**：

- Rank IC 为负时，按 `tree_score` 降序选 Top-K 会系统性偏向未来表现更差的股票。
- 若生产配置仍使用 `sort_by: xgboost`，需要独立重训与验收（见第七节）。

### 4.3 换手率

- 本次约束后半 L1 换手稳定在 **50.00%**（调仓日换手扣费与重叠约束共同作用）。
- 相较旧版周频 ~80% 换手，**摩擦成本显著下降**；全样本成本拖累约 **-1.31%/年**（见第五节）。

### 4.4 执行模型偏差（已修复/已规避）

- `tplus1_open` 路径上曾存在涨停代理误杀问题；本脚本与本次报告统一采用 `**close_to_close`**，绕过该偏差来源。
- 若需仿真「开盘成交」，建议在调仓日粒度单独启用可交易性过滤，而非对全体交易日做 open-to-open 涨停代理。
