## P0：紧急修复

### P0-1 · M7 pitfix 报告行业极度集中（100% 非银金融）`[x]`

#### 问题描述

`docs/monthly_selection_m7_recommendation_report_pitfix_2026_05_03_2026-05-03.md` 显示：

```
Top20 推荐：全部 20 只均属 非银金融/证券Ⅱ（industry_share = 1.0）
华泰证券、申万宏源、兴业证券、招商证券… 直至信达证券
```

而同一模型（M6 xgboost_rank_ndcg, U2_risk_sane）在 2026-04-30 版本（无 pitfix）推荐名单中行业分布正常（公用事业、环保、有色金属等 11 个行业）。

**根本原因分析**：

`feature_fundamental_asset_turnover_z` 在证券行业的截面 z-score 异常高（+3.17 ～ +5.0），是本次推荐高度集中的主导因子。PIT 修复后，证券行业的基本面报告期更准确地对齐到信号日，导致 `asset_turnover` 的相对排名突变——证券公司的资产周转率（利息收入/总资产口径）在 2026-03-31 的截面上被模型系统性高估。

这本质上是两个子问题：
1. **模型层**：M6 xgboost_ranker 对 `asset_turnover` 的权重过高，缺乏行业内相对中性化。
2. **报告层**：`run_monthly_selection_report.py` 没有在推荐输出时强制执行行业上限。

#### 涉及文件

- `scripts/run_monthly_selection_report.py`（报告输出层无行业 cap）
- `src/pipeline/monthly_ltr.py` — `_train_predict_xgboost_ranker`
- `src/features/fundamental_factors.py` — `asset_turnover` 特征构造

#### 修改方案

**Step 1：报告层强制行业上限（最快可用）**

在 `run_monthly_selection_report.py` 的推荐名单生成函数中，在按 `score` 降序后、截取 Top-K 之前，加入贪心行业 cap 过滤：

```python
def apply_industry_cap(
    ranked: pd.DataFrame,
    *,
    top_k: int,
    max_industry_share: float = 0.30,   # 单行业最多 30%（即 Top20 中最多 6 只）
    industry_col: str = "industry",
    score_col: str = "score",
) -> pd.DataFrame:
    """
    贪心选取：按 score 降序遍历，当某行业已达 max_industry_share * top_k 时跳过。
    保证最终推荐集大小 <= top_k。
    """
    cap = max(1, int(np.floor(max_industry_share * top_k)))
    industry_counts: dict[str, int] = {}
    selected = []
    for _, row in ranked.sort_values(score_col, ascending=False).iterrows():
        ind = row[industry_col]
        if industry_counts.get(ind, 0) >= cap:
            continue
        selected.append(row)
        industry_counts[ind] = industry_counts.get(ind, 0) + 1
        if len(selected) >= top_k:
            break
    return pd.DataFrame(selected)
```

在 `M9_integrity` 检查之前、输出推荐表之前调用，并在报告中记录 `industry_cap_applied: true` 和实际 cap 值。

**Step 2：模型层行业内因子标准化**

在 `monthly_ltr.py` 中，对基本面因子（特别是 `asset_turnover`）在训练前做**行业内** z-score 标准化，而非全截面 z-score：

```python
def industry_neutral_zscore(
    df: pd.DataFrame,
    feature_cols: list[str],
    *,
    date_col: str = "signal_date",
    industry_col: str = "industry_level1",
) -> pd.DataFrame:
    """
    对每个 (signal_date, industry) 分组，将 feature_cols 做 z-score。
    消除行业间特征分布差异，防止高 asset_turnover 行业（如证券）系统性排名靠前。
    """
    out = df.copy()
    for col in feature_cols:
        out[f"{col}_ind_z"] = (
            out.groupby([date_col, industry_col])[col]
            .transform(lambda s: (s - s.mean()) / (s.std(ddof=0) + 1e-12))
            .clip(-5, 5)
        )
    return out
```

将 `_z` 后缀特征替换为 `_ind_z` 进入 M6 训练，同步更新 `FeatureSpec` 中的 `feature_cols`。

**Step 3：验收标准**

- 新增单测 `test_monthly_selection_report.py::test_apply_industry_cap_reduces_concentration`：
  构造一个全部属于同一行业的打分 DataFrame，断言 `apply_industry_cap(max_industry_share=0.30)` 后单行业占比 ≤ 30%。
- 对 `2026-03-31` 信号日重跑 M7 pitfix 报告，确认 Top20 中单行业数量 ≤ 6。
- 在 M9 integrity gate 中新增检查：
  ```python
  {"check": "max_industry_share_le_30pct", "pass": top20_max_share <= 0.30, ...}
  ```

**Step 2：特征层防御**

在 `fundamental_factors.py` 中，对 `DEFAULT_FUNDAMENTAL_COLS` 所有字段加入覆盖率检查：

```python
# 在 pit_safe_fundamental_rows 返回后调用
LOW_COVERAGE_THRESHOLD = 0.10  # 候选池内覆盖率低于 10% 则排除

def filter_low_coverage_cols(
    df: pd.DataFrame,
    cols: list[str],
    *,
    threshold: float = LOW_COVERAGE_THRESHOLD,
    pool_mask: pd.Series | None = None,
) -> tuple[list[str], list[str]]:
    """返回 (active_cols, dropped_cols)，dropped_cols 仅保留 is_missing 标志。"""
    active, dropped = [], []
    check = df[pool_mask] if pool_mask is not None else df
    for col in cols:
        cov = check[col].notna().mean() if col in check.columns else 0.0
        (active if cov >= threshold else dropped).append(col)
    return active, dropped
```

### P1-2 · fund_flow 历史覆盖短导致 walk-forward 训练阶段性泄漏 `[x]`

#### 问题描述

fund_flow 数据从 2023-10 开始覆盖，覆盖率约 10-13%：

```
feature_fund_flow_main_inflow_5d   覆盖率: 12.7%  起始: 2023-10-31
feature_fund_flow_main_inflow_10d  覆盖率: 10.8%  起始: 2023-11-30
```

在 walk-forward 回测中（训练窗口涵盖 2021-01 至 2023-09），fund_flow 特征全部缺失（`is_missing=1`）。

**问题后果**：
1. 2021-2023 训练样本中，`is_missing_feature_fund_flow_*` 恒为 1。模型可能学到"有资金流数据 = 特殊股票 = 好股票"的虚假规律。
2. 2023-10 之后，少数股票有资金流数据，多数没有，`is_missing=0` 的股票在测试集上享有来自训练集偏差的优势。
3. 历史 IC / 分桶 spread 包含了这个阶段性泄漏，导致报告中 fund_flow 的增量贡献被高估。

#### 涉及文件

- `src/pipeline/monthly_multisource.py` — `add_zscore_and_missing_flags`
- `src/data_fetcher/fund_flow_client.py`

#### 修改方案

**Step 1：按数据可用起始日期分期训练**

在 `monthly_multisource.py` 中，对每个 feature family 记录其实际数据起始日期（`family_start_date`）：

```python
FAMILY_DATA_START: dict[str, str] = {
    "price_volume": "2018-01-01",     # 全量历史
    "industry_breadth": "2018-01-01",
    "fundamental": "2018-01-01",
    "fund_flow": "2023-10-01",         # 实际起始日
    "shareholder": "2019-01-01",
}
```

在训练集截取时，当 fold 的训练末尾 `< family_start_date` 时，从该 family 的特征中排除所有 z-score 列（保留 `is_missing` 列但权重设为 0）：

```python
def get_active_features_for_fold(
    all_features: list[str],
    fold_train_end: pd.Timestamp,
    family_start: dict[str, str],
) -> list[str]:
    active = []
    for feat in all_features:
        family = _infer_feature_family(feat)
        start = pd.Timestamp(family_start.get(family, "2000-01-01"))
        if fold_train_end >= start or feat.startswith("is_missing_"):
            active.append(feat)
    return active
```

**Step 2：fund_flow 只在有数据的 fold 作为主特征**

将 fund_flow 的 `_z` 特征仅在 `fold_train_end >= 2023-10-01` 时加入，
is_missing 标志在所有 fold 均可用（作为信息缺失指示）。

**Step 3：增量 delta 报告分拆**

在 M5 增量 delta 报告中，新增分期对比列：

```
fund_flow_contribution_pre_2023_10  (空，无有效数据)
fund_flow_contribution_post_2023_10 (实际贡献)
```

**验收标准**

- 重跑 M5 multisource，fund_flow 增量 delta（`delta_topk_excess_mean`）在 `2023-10 前`和`后`的对比展示在报告中。
- 单测 `test_monthly_selection_multisource.py::test_fund_flow_not_in_pre_2023_folds` 通过。

### P2-1 · feature_fundamental_gross_margin_change_z 截面 z-score 被截尾上限 +4.2 多次出现 `[x]`

#### 问题描述

在 M7 推荐报告中，大量股票的 `feature_contrib` 显示 `feature_fundamental_gross_margin_change_z=+4.2`，
这正好是 `_winsor_zscore` 函数的截尾上限 `clip(-5.0, 5.0)` 前的 quantile 0.99 截点。
说明在该截面日，毛利率变化分布极度右偏，大量股票聚集在上限边界。

此时 `gross_margin_change_z` 对高分组的区分能力几乎为零（都是 +4.2），但模型仍赋予其较高特征重要性。

#### 修改方案

在 `_winsor_zscore` 之后，对 z-score 分布做 **rank-based transform**（秩变换），
将 z-score 的截面排名映射到 [-1, +1] 均匀分布，消除上限堆积效应：

```python
def rank_transform(series: pd.Series) -> pd.Series:
    """
    将 z-score 转换为截面秩分（均匀分布于 [-1, 1]）。
    对极值堆积的 z-score 分布更鲁棒。
    """
    n = series.notna().sum()
    if n < 2:
        return pd.Series(0.0, index=series.index)
    ranked = series.rank(method="average", na_option="keep")
    return ((ranked - 1) / (n - 1) * 2 - 1).clip(-1, 1)
```

在 `add_zscore_and_missing_flags` 后，对所有 `_z` 列可选地应用 rank transform（`M5RunConfig.use_rank_transform: bool = True`）。
