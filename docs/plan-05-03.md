### P0-2 · 因子截面 Winsorization（异常值截尾） [x]

**问题**
`attach_signal_features` 输出的所有 `feature_*` 列未做截尾处理。A 股退市整理期、复牌后的极端收益（±100%）直接进入模型，污染 XGBoost 分裂节点和 OLS 回归系数。`neutralize_size_industry_regression` 的 OLS 残差也因输入 y 未截尾产生粗大误差。

**涉及文件**
- `src/pipeline/monthly_dataset.py` — `attach_signal_features`
- `src/features/neutralize.py` — `neutralize_size_industry_regression`

**实施步骤**

1. 在 `monthly_dataset.py` 中新增工具函数：

   ```python
   def winsorize_features(
       df: pd.DataFrame,
       feature_cols: list[str],
       *,
       date_col: str = "signal_date",
       q_low: float = 0.01,
       q_high: float = 0.99,
   ) -> pd.DataFrame:
       """对每个截面日期，将 feature_cols 中的值截尾至 [q_low, q_high] 分位数。"""
       out = df.copy()
       for col in feature_cols:
           if col not in out.columns:
               continue
           out[col] = out.groupby(date_col)[col].transform(
               lambda s: s.clip(
                   lower=s.quantile(q_low),
                   upper=s.quantile(q_high),
               )
           )
       return out
   ```

2. 在 `attach_signal_features` 末尾、返回之前调用 `winsorize_features(df, FEATURE_COLS)`。

3. 在 `neutralize_size_industry_regression` 的 OLS 部分对 y 同样做 IQR 截尾：

   ```python
   # 在 lstsq 之前
   q1, q3 = np.nanpercentile(y[m], [1, 99])
   y_clipped = np.clip(y, q1, q3)
   beta, _, _, _ = np.linalg.lstsq(X[m], y_clipped[m], rcond=None)
   ```

4. 在基本面因子的股东数据路径（`shareholder_factors.py`）对 `holder_change_rate` 同样应用 ±3σ 截尾，处理除权日跳变。

**验收标准**
- 新增单测 `test_winsorize_features`，断言截尾后 `feature_ret_60d` 最大值 ≤ q99 值。
- 对 2020-05（新冠复牌月）截面，确认 `feature_ret_60d` 极值由 >200% 降至合理范围。

### P0-3 · 协方差矩阵 Ledoit-Wolf 收缩估计 [x]

**问题**
`covariance_diagnostics` 已能检测到病态矩阵（条件数过大），但检测到问题后没有自动触发收缩。20 只股票 × 120 天窗口时，样本协方差矩阵条件数轻易超过 1000，均值–方差优化器权重极度集中或求解不稳定。

**涉及文件**
- `src/portfolio/covariance.py`
- `src/portfolio/optimizer.py` — `risk_parity_weights`, `mean_variance_weights`

**实施步骤**

1. 在 `covariance.py` 中实现自适应收缩函数：

   ```python
   def estimate_covariance(
       returns: np.ndarray,
       *,
       method: str = "auto",  # "sample" | "ledoit_wolf" | "factor" | "auto"
       condition_threshold: float = 1000.0,
   ) -> np.ndarray:
       """
       自适应协方差估计：条件数超过阈值时自动切换至 Ledoit-Wolf。
       """
       from sklearn.covariance import LedoitWolf

       cov_sample = np.cov(returns, rowvar=False)
       if method == "sample":
           return cov_sample

       diag = covariance_diagnostics(cov_sample)
       if method == "auto" and diag["condition_number"] < condition_threshold:
           return cov_sample

       lw = LedoitWolf().fit(returns)
       return lw.covariance_
   ```

2. 在 `optimizer.py` 的 `risk_parity_weights` 和 `mean_variance_weights` 中，将直接使用 `Sigma` 替换为先调用 `estimate_covariance`。

3. 在 `config.yaml` 新增配置项：

   ```yaml
   portfolio:
     covariance_method: "auto"       # sample | ledoit_wolf | auto
     condition_number_threshold: 1000
   ```

4. 扩充 `covariance_diagnostics` 输出，加入 `shrinkage_intensity` 字段（Ledoit-Wolf 的 α 值）。

**验收标准**
- 新增单测：构造病态协方差矩阵（条件数 > 10000），断言 `estimate_covariance(method="auto")` 返回的矩阵条件数 < 1000。
- 月度优化报告中展示协方差矩阵收缩强度，记录到 `research_contract` metadata。

   ```python
       # Barra-style 3 因子分解
       # 因子矩阵 F: (T, K)，特异性方差 D: diag
       beta = np.linalg.lstsq(factor_returns, asset_returns, rcond=None)[0]
       factor_cov = np.cov(factor_returns, rowvar=False)
       specific_var = np.var(asset_returns - factor_returns @ beta, axis=0)
       return beta.T @ factor_cov @ beta + np.diag(specific_var)
   ```

2. 因子选取：市场等权收益（已有 `MARKET_COL`）、市值因子（`log_market_cap`）、动量因子（`feature_ret_60d`）。

**验收标准**
- 单测：因子模型协方差矩阵为正定，条件数低于样本协方差矩阵。

### P2-6 · 换手约束纳入组合优化 [x]

**问题**
`optimizer.py` 在优化时不考虑换手成本约束，只在事后计算成本。月度大换手（> 60%）严重侵蚀收益，但当前框架无法在优化层约束。

**涉及文件**
- `src/portfolio/optimizer.py` — `risk_parity_weights`, `mean_variance_weights`
- `config.yaml`

**实施步骤**

1. 在优化函数签名中增加参数：

   ```python
   def mean_variance_weights(
       mu: np.ndarray,
       Sigma: np.ndarray,
       *,
       prev_weights: Optional[np.ndarray] = None,
       max_turnover: float = 1.0,   # 1.0 = 不约束
       ...
   ) -> tuple[np.ndarray, dict]:
   ```

2. 当 `prev_weights is not None and max_turnover < 1.0` 时，在 scipy 约束中加入：

   ```python
   # 引入辅助变量 z_i = |w_i - w_prev_i|，线性化绝对值约束
   constraints.append({
       "type": "ineq",
       "fun": lambda w: max_turnover - np.sum(np.abs(w - prev_weights)),
   })
   ```

3. 在 `config.yaml` 新增：

   ```yaml
   portfolio:
     max_monthly_turnover: 0.60
   ```

**验收标准**
- 单测：设置 `max_turnover=0.3` 时，优化器输出的权重与前期权重 L1 差 ≤ 0.3。
- 月度回测换手率分布：均值从 ~70% 降至 ~50%，净收益提升（摩擦成本减少）。
