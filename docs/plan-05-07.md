## 2. 当前决策框架

### 2.1 核心问题定义

```text
score(stock, month_end)
-> rank tradable universe
-> select monthly Top-K (默认 20)
```

每个月最后一个交易日 `t`，只使用 `t` 日收盘后已可见的 PIT-safe 数据，在 `t+1` 开盘可买的股票池中排序，标签为 open-to-open 收益。

### 2.2 口径固定值

| 字段 | 当前默认 |
|------|----------|
| `rebalance_rule` | `M` |
| `signal_date` | 每月最后一个交易日 |
| `execution_mode` | `tplus1_open` |
| `label_return_mode` | `open_to_open` |
| `benchmark` | `market_ew_open_to_open` |
| `top_k` | 20（主报告） |
| `cost_bps` | 16（散户基准），压力测试 30/50 |
| `portfolio_method` | 等权；生产前必须满足 indcap3 + redistribution |
| `industry_map` | 申万二级行业 |

### 2.3 当前最重要判断

1. **hard-cap baseline 是生产唯一基准**。NW-t=3.65（p<0.001），Bootstrap CI 下界 +0.75%，IR=0.51，`promoted_registry.json` 已写入，`human_confirmed=true`。
2. **OOS 积累是当前最高优先级**。唯一月度必须操作，任何研究工作不得影响每月 SOP 执行。
3. **M13-A ExtraTrees+concept 不纳入 promotion**。delta=+0.08%（p=0.87），不显著。概念因子方向转由 M13-B 个股绑定路线验证。
4. **M14 LHB 保持数据更新，不独立 promote**。LHB+ExtraTrees 对 PV baseline 有增量，但未超 hard-cap baseline（−0.19%）。LHB 因子作为低权重辅助特征保留，待下轮 ensemble 研究中评估。
5. **北向个股数据无法用于 2024-08 后信号**。监管截断为监管政策而非技术问题，2024-08 后仅市场级汇总可用，北向因子重构为 regime 特征（非个股 Alpha）。
6. **M13-B 是下一个最高潜力研究方向**。个股-概念绑定一旦数据可用，预计增量高于 ExtraTrees+concept 的 P5-V1（板块等权 breadth），因为绑定精度更高。
7. **生产边界不变**：任何研究候选通过完整 gate（NW-t≥2.5, Bootstrap CI>0, IR≥0.5, 30bps after-cost>0, OOS≥1月, buy-fail 通过）前，不写入 `config.yaml.example` 或 registry。

## 4. M15：生产自动化 & 月度 SOP 脚本 ✅（2026-05-07 完成）

**目标**：将散落在多个脚本的月度操作合并为一个 `scripts/run_monthly_production.py`，减少人为操作错误。

### 4.1 脚本设计

```python
# scripts/run_monthly_production.py
# 用法：python scripts/run_monthly_production.py --month 2026-05

# 内部调用顺序：
# 1. fetch_only.py（增量更新日线）
# 2. run_monthly_selection_report.py（生成 Top-20，自动选信号日；
#    脚本内部自动调用 record_oos_from_m7_report 写入预测 + 回填上月实现值）
# 3. run_monthly_benchmark_suite.py（更新成本压力表）
# 4. 告警检查（OOS DuckDB 查询 / 候选池宽度 / next_trade_date / LHB 陈旧）
# 5. 输出摘要：信号日、Top-20 名单、OOS 历史表
```

参数：`--month`（必填）、`--skip-fetch`、`--config`（默认 hardcap baseline）、`--dry-run`

关键实现细节：
- `config_id`、`cost_bps`、`model`、`candidate_pool`、`top_k` 均从 promoted config JSON 读取，非硬编码
- OOS 告警直接从 DuckDB `oos_tracking` 表查询，不再依赖 `oos_tracking.json` 文件
- LHB 新鲜度检查查询 `a_share_lhb_daily` 最新 `data_date`

### 4.2 告警机制

- OOS 连续 N 个月超额 < −1%：打印 `[WARNING] OOS 连续失效告警，请人工复核`
- 候选池通过标的 < 2000：打印 `[WARNING] 候选池异常收窄`
- 新信号日 `next_trade_date` 为空：抛出 `ValueError`（不允许静默失败）
- LHB 数据最新日期落后今日 >30 天：打印 `[WARNING] LHB 数据可能需要更新`

features_concept_v2 = [
    "concept_member_count",          # 个股所属概念板块数（广度代理）

### 5.4 M5 Gate 结果（2026-05-07）

**数据准备**：
- 回填 `a_share_concept_daily_em`（EM 概念日线，BK 代码）：486 概念 / 535,653 行 / 2020-01-02 ~ 2026-05-07，0 失败

- 修复 `concept_factors.py`：EM 日线与 membership 代码体系统一（BK codes）

**M5 WF 结果**（U1 + Top-20 @16bps，39 个测试月）：

| 模型 | After-Cost 超额 | Rank IC | Rank IC IR | vs PV Delta 超额 | vs PV Delta Rank IC |

**Gate 判定**：

| Gate | 阈值 | ExtraTrees | ElasticNet |
|------|------|-----------|------------|
| Rank IC delta > 0.005 | > 0.005 | ❌ 0.0027 | ✅ 0.0086 |
| WF excess delta ≥ 0（3 年） | ≥ 0 | ✅ +0.97%（2023: −1.4%, 2024: +0.7%, 2025: +2.5%, 2026: +4.9%） | ❌ −0.20% |

**特征重要性**（ExtraTrees top-20）：
- `feature_concept_member_count_z`：**0.0665**（第 5 重要，仅次于量价核心）

**结论**：
1. **无单一模型同时通过两道 gate**。ExtraTrees 超额 delta 强（+0.97%）但 Rank IC 增量未达标；ElasticNet Rank IC 达标但无法转化为超额收益。

3. **⚠️ 关键限制**：伪历史快照意味着 membership 特征在回测中使用了未来信息（当前成分股回填到 2021 年）。真实 PIT-safe 验证需 ≥6 月累积真实月度快照后重新评估。
4. **有条件通过**（conditional pass）：经济显著性明确（+0.97%/月 after-cost），方向性稳定改善（2023→2026 delta 单调递增），但需 PIT-safe 复检后决定是否纳入 promotion gate。

## 6. M13-C：北向资金因子重构

**当前状态（2026-05-07）**：个股北向持股数据受 2024-08-16 监管截断，无法用于 2024-08 后的信号生成。333 只标的有历史数据（2021–2024-08），市场级汇总数据（`stock_hsgt_hist_em`）有完整历史及当日更新。

**重构策略**：将北向因子从个股 Alpha 信号重定位为 regime 特征。

| 数据段 | 策略 |
|--------|------|
| 回测期（2021~2024-08） | 个股级北向持股变化作为 M5 特征，333 只覆盖标的 |
| 2024-08 后 | 市场级北向汇总净买入作为 regime 特征（类 RSI 信号），而非个股 Alpha |

### 6.1 市场级北向 regime 特征设计

```python
features_northbound_regime = [
    "north_net_inflow_1m",      # 近 1 月北向净流入总额（市场级）
    "north_net_inflow_3m",      # 近 3 月北向净流入总额
    "north_inflow_zscore_6m",   # 近 1 月净流入相对 6 月均值的 z-score
    "north_consecutive_outflow_days",  # 连续净流出天数（风险 regime 信号）
]
```

这些特征作为市场层面 regime 信号（类似现有的 strong_up/down 分类），辅助个股打分，而非直接作为个股 Alpha 因子。

### 6.2 验证步骤

```bash
# Step 1：构建市场级北向特征
python scripts/load_northbound.py --mode market_regime --start-date 2021-01-01

# Step 2：M5 regime 增量 delta 测试
python scripts/run_monthly_selection_multisource.py \
  --config config.yaml.backtest \
  --extra-features northbound_regime \
  --output-stem m13c_northbound_regime_delta
```

Gate：Rank IC 提升 > 0.003（较低门槛，因为 regime 特征为辅助特征）。

### 6.3 M5 Gate 结果（2026-05-07）—— ❌ FAIL

**数据发现**：市场级北向数据（`stock_hsgt_hist_em`）同样受 2024-08-16 监管截断影响。API 返回日期到 2026-05-07 但所有财务列（净买额、买入/卖出成交额等）均为 NaN。2264 行（2014-11-17 ~ 2024-08-16）。

**特征设计修正**：市场级 regime 特征对所有个股同截面同值，`add_zscore_and_missing_flags` 截面 z-score 后全为 0。V2 修复为使用原始值（不做截面 z-score）。

**M5 WF 结果**（U1 + Top-20 @16bps，39 个测试月）：

| 模型 | After-Cost 超额 | Rank IC | vs PV Delta 超额 | vs PV Delta Rank IC |
|------|----------------|---------|-----------------|---------------------|
| Plus Northbound Regime ExtraTrees | 1.22% | 0.1000 | **−0.02%** | **−0.0033** |
| PV ExtraTrees（对照）| 1.24% | 0.1033 | — | — |
| Plus Northbound Regime ElasticNet | 1.56% | 0.1030 | **−0.08%** | **−0.0007** |
| PV ElasticNet（对照）| 1.64% | 0.1038 | — | — |

**Gate 判定**：

| Gate | 阈值 | ExtraTrees | ElasticNet |
|------|------|-----------|------------|
| Rank IC delta > 0.003 | > 0.003 | ❌ −0.0033 | ❌ −0.0007 |

**特征重要性**（ExtraTrees，4 个 regime 特征合计 ~17.7% importance）：
- `feature_north_net_inflow_3m`：0.0691
- `feature_north_inflow_zscore_6m`：0.0470
- `feature_north_consecutive_outflow_days`：0.0310
- `feature_north_net_inflow_1m`：0.0297

**结论**：
1. 市场级 regime 特征虽被模型使用（ExtraTrees 合计 ~17.7% 重要性），但未转化为正向增量——Rank IC 和超额 delta 均为负。
2. **根本原因**：零截面方差——同月所有个股获得相同 regime 值，模型无法通过 regime 特征区分个股。ExtraTrees 虽能用 regime 特征做"月度开关"，但这类按月的 split 更可能是过拟合而非稳定 regime 信号。
3. 北向数据（个股级+市场级）均已验证无法在 2024-08 后使用。北向因子全面冻结。
4. **建议**：M13-C 归入停止清单。regime 特征的截面方差问题本质性（非数据量或模型选择问题），无重试价值。
