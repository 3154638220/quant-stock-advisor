### 0.2 16 bps 成本下的实际收益（M8 natural 候选，ElasticNet）

毛月收益（gross）= **1.56%/月**，月换手率约 90%。

```
你的月成本拖累 = 90% × 16 bps = 0.144%/月
你的净月收益   = 1.56% - 0.144% ≈ 1.42%/月

机构 30bps 情景 = 1.56% - 0.27% ≈ 1.29%/月
机构 50bps 情景 = 1.56% - 0.45% ≈ 1.11%/月
```

**结论**：你每月多拿约 0.13%（vs 机构 30bps 情景），年化多约 1.5pp。"换手高"对你是中性的，不是风险。

## 9. P8：Oracle 条件 IC Gate

### 9.1 为什么要建这个 gate

M5 增量 delta 只看全截面 Rank IC，不能直接反映新数据是否帮助选中真正的好股票。

### 9.2 新增评估指标

```python
def oracle_conditional_ic(factor, oracle_topk_mask):
    """
    在 Oracle Top-20 子集内计算 factor 与未来收益的 IC
    衡量：这个因子对"真正好的股票"有多强的辨别力
    """
```

### 9.3 新数据准入标准

| 条件 | 要求 |
|------|------|
| 全截面增量 Rank IC delta | > 0.002 |
| Oracle 条件 IC 均值 | > 0.03 |
| Oracle Top-20 overlap 提升 | ≥ 2pp |

三条全部通过才进入 M5 主线，否则只作辅助特征。

## 12. 散户执行提示

月度换仓的实际操作要点：

| 时间 | 动作 |
|------|------|
| 每月最后一个交易日收盘后 | 运行 M7 报告，生成 Top-20 名单 |
| 当晚或次日 9:15–9:25 | 集合竞价挂单，尽量以开盘价成交 |
| 次日 9:30 开盘后 | 确认成交；涨停买入失败时等价补入其他候选 |
| 同时卖出上月持仓 | 同样优先集合竞价（9:15–9:25）卖出 |

**集合竞价执行**是让你的成本口径与回测口径（T+1 开盘价）对齐的关键，实际偏差 0.1–0.3% 在可接受范围内。
