## 0. 背景与核心假设

### 观察来源

东方财富日线图的「多空趋势线」颜色状态具有高度自相关性：一旦变红（多头），往往连续多日维持红色；一旦变绿（空头），连续绿色同样常见。基于此信号做单股回测的视觉核验显示：变红买入、变绿卖出的策略能够有效捕捉趋势段，同时因信号持续性高而避免了高换手。

### 核心假设

| 假设 | 可检验性 |
|------|---------|
| A1：多空趋势线的「红/绿」状态在截面上对下月收益具有正向预测力 | IC Gate：单因子 Rank IC |
| A2：趋势持续天数（连红天数）比单日状态具有更稳定的 IC | IC Gate：子因子对比 |
| A3：将趋势状态作为候选池过滤条件，能提升 M8 月超额 | M8 Baseline Gate |
| A4：用指数级别的趋势状态作为大盘 Regime 补充信号，能改善 M8 策略的大盘判断 | M8 Regime Gate |

### 与现有体系的定位

本因子族（暂称 `trend_persistence`）属于 **价量衍生**，但与现有 `price_volume` 族的关键区别在于：

- `price_volume` 族（动量、ATR、RSI 等）是连续值因子，在截面上直接打分
- `trend_persistence` 族捕捉的是**状态机的持续性**：信号是离散的（多/空），核心信息是状态持续了多久、翻转的代价有多大

这与 `reversal_volume` 族（反转+量能异常）在机制上相反——后者试图捕捉均值回归，前者试图顺势延伸。两者是竞争假设，不应同时入模。

## 1. 信号复现：多空趋势线的近似构造

东方财富的多空趋势线为专有指标，无公开算法。以下给出三种从 `a_share_daily` 表可直接计算的近似实现，按复杂度排序。

### 方案 A（基准复现）：EMA 双线交叉

```
快线 = EMA(close, 12)
慢线 = EMA(close, 26)
多空状态 = 1（bull）if 快线 > 慢线 else 0（bear）
```

参数说明：12/26 是 MACD 的标准参数，东方财富多空线本质上是对 DIF（EMA12 - EMA26）的零线判断。当 DIF > 0 时趋势线为红，DIF < 0 时为绿。这与截图中可见的 MA 数值参数（202151 系列）吻合。

### 方案 B（增强）：价格位置 + EMA 交叉双重确认

```
状态 = 1 if (EMA12 > EMA26) AND (close > EMA26) else 0
```

双重确认减少假突破，代价是信号更滞后。

### 方案 C（参数扫描版）：多组 MA 交叉多数投票

```
v1 = sign(MA5 - MA20)
v2 = sign(MA10 - MA30)
v3 = sign(EMA12 - EMA26)
状态 = 1 if (v1 + v2 + v3) > 0 else 0
```

参数更鲁棒，避免过拟合到单一参数组。

### 验证策略

用方案 A 作为基准复现，在至少 20 只典型 A 股（覆盖不同市值、行业、牛熊周期）上与东方财富 App 截图对比，确认颜色翻转日期吻合率 > 85%。若不吻合，尝试方案 B 或调整 EMA 参数。

## 2. 因子构造

在月末信号日（`signal_date`）计算截面因子。所有因子均为 PIT-safe（仅用 `trade_date ≤ signal_date` 的数据）。

### 2.1 拟构造因子列表

| 因子列名 | 描述 | 方向 |
|---------|------|------|
| `feature_trend_bull_state` | 信号日当天是否为多头（1/0） | +1（多头=正向） |
| `feature_trend_streak_days` | 当前状态连续天数（多头为正，空头为负） | +1（连续多头天数越多越好） |
| `feature_trend_bull_ratio_20d` | 过去 20 交易日中多头天数占比 | +1 |
| `feature_trend_bull_ratio_60d` | 过去 60 交易日中多头天数占比 | +1 |
| `feature_trend_flip_days_ago` | 距上次状态翻转的天数（越大越稳定） | +1 |
| `feature_trend_ema_spread` | EMA12 - EMA26（归一化为 close 的百分比） | +1（连续动量强度） |

前两个因子直接对应用户观察到的交易逻辑；后几个是衍生的持续性量化指标。

### 2.2 候选池级别的趋势过滤（独立测试路线）

除了作为评分因子，趋势状态也可作为**候选池前置过滤条件**：

```python
# 仅保留信号日处于多头状态的股票
candidate_df = candidate_df[candidate_df["feature_trend_bull_state"] == 1]
```

这与 U0/U1/U2 流动性过滤的逻辑类似，是在打分前剔除信号不利的股票。

该过滤效果需单独回测：过滤后候选池收缩（典型情况下多头/空头各占约 50% 截面），需确认不会引入幸存者偏差或过度集中于特定市场状态。

### 2.3 大盘级别趋势信号（补充 Regime）

用指数日线（`510300` 或 `market_ew_proxy`）计算相同的多空趋势状态，作为 `RegimeConfig` 的补充输入：

- `index_trend_bull`：当前指数多头/空头状态
- `index_trend_streak_days`：指数连续多头/空头天数

这可以在 `src/market/regime.py` 的 `classify_regime()` 中叠加，作为熊市判断的辅助信号，与现有短期收益阈值机制并行而非替代。

## 3. 验证路径

遵循项目两阶段 Gate 体系：**IC Gate (Stage A) → M8 Baseline Gate (Stage B)**。

### Stage A：IC Gate

**目标**：验证单因子的截面预测能力，决定哪些子因子值得进入 Stage B。

**命令**：

```bash
python scripts/run_monthly_selection_multisource.py \
  --families price_volume,trend_persistence \
  --output-prefix w7_trend_stage_a_$(date +%Y_%m_%d) \
  --ml-models elasticnet,extratrees \
  --model-n-jobs 0
```

**通过标准**（与 W1 Phase 对齐）：

| 指标 | 阈值 |
|------|------|
| 个体 Rank IC 均值 | ≥ 0.01（约为 price_volume 族中位数水平） |
| IC IR | ≥ 0.3 |
| 正向期数占比 | ≥ 55% |
| `price_volume + trend_persistence` vs `price_volume only` 的 Rank IC delta | > 0 |

若 `feature_trend_bull_state` 单因子不通过（Rank IC ≈ 0），优先检查：
1. 因子方向是否错误（多头选股信号可能是反向的，取决于截面）
2. 候选池过滤后截面数量是否过小（< 100 只）
3. 历史数据中多头/空头分布是否极度不均衡

### Stage B：M8 Baseline Gate

**仅当 Stage A 通过后执行。**

**实验组设计**：

```bash
# 基准复现（生产默认）
python scripts/run_monthly_selection_concentration_regime.py \
  --families industry_breadth,fund_flow,fundamental,quality \
  --output-prefix w7_m8_baseline_rerun_$(date +%Y_%m_%d)

# 实验组：加入趋势持续性因子
python scripts/run_monthly_selection_concentration_regime.py \
  --families industry_breadth,fund_flow,fundamental,quality,trend_persistence \
  --output-prefix w7_m8_plus_trend_$(date +%Y_%m_%d)

# 实验组（候选池过滤版）：在 U1 基础上叠加多头过滤
# 需要在 run_monthly_selection_concentration_regime.py 中新增 --trend-filter 参数
python scripts/run_monthly_selection_concentration_regime.py \
  --families industry_breadth,fund_flow,fundamental,quality \
  --trend-filter bull_state \
  --output-prefix w7_m8_trend_filter_$(date +%Y_%m_%d)
```

**通过标准**：

| 指标 | 阈值 |
|------|------|
| after-cost 月超额 | ≥ 生产 baseline（当前 2.15%，@16bps） |
| baseline delta | ≥ 0 |
| NW-t | ≥ 2.5 |
| Bootstrap 95% CI 下界 | > 0 |
| top-K vs next-K spread | > 0 |

**关键对比**：Stage B 需同时对比**因子入模**（trend_persistence 作为 feature）和**候选池过滤**（trend_filter 作为 gate）两条路线，取效果更好者。

## 5. 候选池过滤的潜在风险与应对

这是本计划与过去 W 系列因子研究最大的不同点：**候选池过滤会改变截面结构**，需要额外注意以下风险。

| 风险 | 说明 | 应对 |
|------|------|------|
| **市场状态偏差** | 多头过滤在牛市中大量保留股票，在熊市中过滤掉大多数，导致策略在熊市时持股极少甚至空仓 | 检查各年度的候选池过滤后样本量；空仓期处理应与 `idle` / `redistribute` 对齐 |
| **行业集中度** | 牛市中多头股票可能高度集中在强势行业，与 `indcap3` 行业分散约束产生冲突 | Stage B 中保留 `indcap3` 约束，观察行业约束的 binding 频率 |
| **回测高估** | 多头状态本身与短期收益高度相关（趋势延伸），IC Gate 可能虚高 | IC 计算窗口与入样筛选窗口应严格隔离，使用 t+1 open-to-open 标签 |
| **换手影响** | 多空状态翻转频繁时，本月选出的股票到下月初已变空头 | 统计月末多头股票在次月调仓日仍维持多头的比例（需 > 70%） |

## 6. 停止条件

以下任一情况触发该研究路线中止，不进入后续阶段：

1. T1 单股视觉验证吻合率 < 85%，且尝试方案 B/C 后仍无改善 → 信号复现失败，记录 API 限制，转为直接爬取东方财富自定义指标
2. Stage A 全部 6 个子因子 Rank IC 均 < 0.005，且候选池过滤路线也无改善
3. Stage B 候选池过滤后月均候选股数 < 60 只（行业约束下持仓不可分散）
4. Stage B after-cost 超额低于当前 baseline 2.15% 超过 0.3%（负向拖累）
