# 项目算法建模详解

更新时间：`2026-05-03`

本文档以当前仓库实现为准，说明项目从数据、特征、标签、截面排序、组合构建到回测治理的完整建模口径。当前系统不是预测某只股票的精确涨幅，而是在给定交易口径下学习一个更可靠的截面排序函数。

当前项目只维护月频选股任务：

| 任务 | 频率 | 入口 | 建模对象 |
| --- | --- | --- | --- |
| 月度选股研究 | 月频/月末 | `scripts/run_monthly_selection_*.py` + `src/pipeline/` | 月末可交易股票池中的未来 1 个月 market-relative Top-K 排序 |

截至当前仓库状态，`configs/promoted/promoted_registry.json` 中 active promoted 配置为 `monthly_selection_u1_top20_indcap3_hardcap_baseline`。本文描述的是研究与月度筛选系统，不是自动交易策略承诺。

---

## 1. 建模目标

设交易日或信号月末为 `t`，股票集合为 `U_t`，股票为 `i in U_t`。系统学习一个打分函数：

```text
s(i,t) = f(x(i,t))
```

其中 `x(i,t)` 是在 `t` 日收盘后已知的特征。月度选股流程是：

```text
Data -> PIT Features -> Candidate Pool -> Cross-sectional Score
     -> Top-K Selection -> Portfolio / Research Report -> Backtest
```

月度链路重点关注 PIT-safe、open-to-open、walk-forward 和可复现实验报告。

### 1.1 实现模块映射

当前实现采用“稳定脚本入口 + `src/` 核心模块”的边界：`scripts/run_*.py` 负责 CLI、路径解析、文件 I/O、报告落盘和 manifest；可复用算法逻辑放在 `src/pipeline/`、`src/reporting/`、`src/research/`。

| 主题 | 核心模块 | 主要职责 |
| --- | --- | --- |
| 月度数据集 | `src/pipeline/monthly_dataset.py` | 月末信号日、信号特征、T+1 可买性、候选池、月度 open-to-open 标签 |
| 月度 baseline helper | `src/pipeline/monthly_baselines.py` | M4/M5/M6 共用的有效样本过滤、截面打分基列、rank pct 分数、sklearn/XGBoost 训练预测 |
| 多源特征 | `src/pipeline/monthly_multisource.py` | 行业宽度、资金流、PIT 基本面、股东户数特征，M5 增量评分 |
| 月度 LTR | `src/pipeline/monthly_ltr.py` | 月度 query 分组、relevance、XGBoost ranker、top20 calibrated 与 ensemble |
| 月度报告 | `src/reporting/monthly_report.py` | 最新信号月选择、full-fit 推荐打分、M9 特征覆盖率策略 |
| 正式回测 | `src/pipeline/backtest_runner.py` | 因子计算、因子缓存、截面打分、Top-K 权重、市场等权基准、Regime 调权、动态 IC 权重 |
| P1 标签 | `src/pipeline/label_builder.py` | daily proxy / monthly investable 标签、可投资收益面板、调仓日选择 |
| gates 与 manifest | `src/research/gates.py`、`src/research/manifest.py` | 共享候选池规则、标签列名、预过滤器、研究身份与标准 manifest |
| Markdown 报告 | `src/reporting/markdown_report.py` | 表格格式化、JSON-safe 序列化、项目相对路径与通用报告片段 |

历史脚本和测试中仍存在从 `scripts/` 导入的兼容路径；新维护点应优先落在上表模块。核心模块保持单向依赖：`scripts/` 可以调用 `src/`，但 `src/pipeline/`、`src/reporting/`、`src/research/` 不应反向导入 `scripts/`。

## 2. 数据与记号

对股票 `i` 在交易日 `t`：

| 记号 | 含义 |
| --- | --- |
| `O(i,t)` | 开盘价 |
| `H(i,t)` | 最高价 |
| `L(i,t)` | 最低价 |
| `C(i,t)` | 收盘价 |
| `V(i,t)` | 成交量 |
| `A(i,t)` | 成交额 |
| `tau(i,t)` | 换手率 |

所有信号只允许使用 `t` 及以前的数据。基本面、资金流、股东数据进入模型前必须按可获得日期或 `availability_lag_days` 做 point-in-time 处理；不允许把未来公告或未来修订信息泄漏到历史训练样本。

## 3. 标签与执行口径

### 3.1 月度 open-to-open 标签

月度研究在每月最后一个交易日 `m` 产生信号，下一交易日开盘执行。若 `R_o2o(i,d)` 表示从 `d` 日开盘到下一个交易日开盘的收益，则月度标签近似为：

```text
y_1m(i,m) = product(1 + R_o2o(i,d)) - 1
```

其中 `d` 从信号日后的第一个 open-to-open 区间开始，到下一次月末信号日开盘结束；不再包含下一信号日到下一月首个交易日开盘的隔夜区间。月度数据集同时生成：

| 标签列 | 含义 |
| --- | --- |
| `label_forward_1m_o2o_return` | 未来 1 个月 open-to-open 原始收益 |
| `label_forward_1m_excess_vs_market` | 原始收益减同月市场等权收益 |
| `label_forward_1m_industry_neutral_excess` | 原始收益减同月行业等权收益 |
| `label_future_return_percentile` | 同月截面未来收益百分位 |
| `label_future_return_quantile` | 同月截面未来收益分桶 |
| `label_future_top_20pct` / `top_10pct` | 未来 top bucket 分类标签 |
| `label_future_bottom_20pct` | 未来底部风险标签 |

月度主线优先优化 market-relative / industry-neutral top-bucket ranking，而不是单票收益的点预测。

## 4. 候选池建模

候选池是准入层，不是 alpha 层。它的职责是排除不可交易、数据无效和极端风险标的，让排序模型面对一个宽而合理的研究空间。

### 4.1 候选池硬过滤工具

`prefilter_stock_pool()` 可在研究诊断中剔除：

- ST / `*ST`，前提是行情中有名称列。
- 近几日涨跌停次数过多的股票。
- 当日换手率处于市场极低或极高分位的股票。
- 当前价格处于长窗口绝对高位的股票。

### 4.2 月度候选池

月度 M2 数据集固定输出三个版本：

| pool | 数学含义 | 工程规则 |
| --- | --- | --- |
| `U0_all_tradable` | `valid_bar and buyable(t+1 open)` | 当前 OHLCV 有效，且次日开盘不是停牌近似或一字涨停不可买 |
| `U1_liquid_tradable` | `U0 and liquid and enough_history` | U0 + 最短历史长度 + 20 日成交额门槛 |
| `U2_risk_sane` | `U1 and not extreme_risk` | U1 + 排除极端涨跌停路径、极端波动/换手和绝对高位 |

当前 active promoted 方法默认使用 `U1_liquid_tradable`。候选池本身不提供 alpha，只定义可交易和风险准入边界。

## 5. 日线基础因子

月度链路通过 `src/features/tensor_base_factors.py` 和 `src/features/intraday_proxy_factors.py` 对宽表张量 `(num_symbols, num_days)` 批量计算日线特征，再在月末信号日截面取值。

### 5.1 收益与动量

单日收益：

```text
r(i,t) = C(i,t) / C(i,t-1) - 1
```

`N` 日动量：

```text
momentum_N(i,t) = C(i,t) / C(i,t-N) - 1
```

短期反转：

```text
short_reversal_N(i,t) = -momentum_N(i,t)
```

`recent_return` 使用短窗口动量，在线性配置中通常给负权重，用于抑制近端过热。

### 5.2 RSI

项目使用 Wilder RSI。令：

```text
delta_C(i,t) = C(i,t) - C(i,t-1)
G(i,t) = max(delta_C(i,t), 0)
D(i,t) = max(-delta_C(i,t), 0)
```

Wilder 平滑：

```text
avg_G_t = ((P-1) * avg_G_{t-1} + G_t) / P
avg_D_t = ((P-1) * avg_D_{t-1} + D_t) / P
RSI_t = 100 - 100 / (1 + avg_G_t / (avg_D_t + eps))
```

`rsi_mode=level` 将高 RSI 解释为趋势强；`mean_revert` 使用 `50 - RSI` 表达均值回归。

### 5.3 波动、ATR 与尾部风险

实现波动率：

```text
realized_vol_W(i,t) = std(r(i,t-W+1), ..., r(i,t))
```

真实波幅：

```text
TR_t = max(H_t - L_t, abs(H_t - C_{t-1}), abs(L_t - C_{t-1}))
ATR_t = ((P-1) * ATR_{t-1} + TR_t) / P
```

窗口内最大单日跌幅：

```text
max_single_day_drop_W(i,t) = min(r(i,t-W+1), ..., r(i,t))
```

注意该值越负，近期尾部风险越强；线性权重的正负要结合截面 z-score 和业务语义解释。

### 5.4 量价结构

换手率滚动均值：

```text
turnover_roll_mean_W(i,t) = mean(tau(i,t-W+1), ..., tau(i,t))
```

量价相关：

```text
vol_ret_corr_W(i,t) = corr(log(1 + V), r) over last W days
```

波动/换手交叉：

```text
vol_to_turnover(i,t) = realized_vol_W(i,t) / (turnover_roll_mean_W(i,t) + eps)
```

成交量偏度使用窗口内 `log(1+volume)` 的三阶标准化矩。

### 5.5 价格位置、市值代理与涨跌停路径

均线乖离：

```text
BIAS_N(i,t) = C(i,t) / MA_N(i,t) - 1
```

区间位置：

```text
price_position_W(i,t) =
    (C(i,t) - min(C over W)) / (max(C over W) - min(C over W) + eps)
```

流通市值代理：

```text
log_market_cap = log(close * volume / (turnover / 100 + eps))
```

`limit_move_count` 统计窗口内 `abs(daily_return)` 接近对应板块涨跌停幅度的次数。

### 5.6 周线 KDJ

`weekly_kdj_from_daily()` 将日线聚合到最近已完成周线，再把 K、D、J 对齐回日线。当前可用输出：

- `weekly_kdj_k`
- `weekly_kdj_d`
- `weekly_kdj_j`
- `weekly_kdj_oversold`
- `weekly_kdj_oversold_depth`
- `weekly_kdj_rebound`

这类因子用于捕捉周线级别的超卖与反弹确认。

### 5.7 K 线结构代理因子

在没有分钟数据时，用日线 OHLCV 的内部结构近似日内行为：

| 因子 | 定义/含义 |
| --- | --- |
| `intraday_range` | `(high - low) / open`，日内振幅 |
| `upper_shadow_ratio` | `(high - max(open, close)) / (high - low)`，上影线压力 |
| `lower_shadow_ratio` | `(min(open, close) - low) / (high - low)`，下影线支撑 |
| `close_open_return` | `close / open - 1`，尾盘相对开盘强弱 |
| `overnight_gap` | `open(t) / close(t-1) - 1`，隔夜跳空 |
| `tail_strength` | `close_open_return` 滚动均值 |
| `volume_price_trend` | `sum(log(1+volume) * daily_return)` |
| `intraday_range_skew` | 日内振幅序列偏度 |

这些因子主要用于描述结构和风险，不应单独被解释为稳定买入理由。

## 6. 多源特征

### 6.1 PIT 基本面

默认基本面列包括：

```text
pe_ttm, pb, ev_ebitda, roe_ttm, net_profit_yoy,
gross_margin_change, gross_margin_delta,
debt_to_assets_change, ocf_to_net_profit, ocf_to_asset,
asset_turnover, net_margin_stability,
northbound_net_inflow, margin_buy_ratio
```

`preprocess_fundamental_cross_section()` 对基本面做：

1. 按日期截面缩尾。
2. 可选市值/行业回归中性化。
3. 按截面中位数填补缺失，兜底为 0。

月度链路会用 `availability_lag_days` 约束可得性，并按信号日读取 PIT 快照。正式回测的 PIT 基本面对齐还会过滤可疑财报行：除日线估值源 `stock_value_em` 外，公告日必须晚于报告期（`announcement_date > report_period`），避免把报告期当作可得日造成未来信息泄漏。

### 6.2 资金流与股东特征

月度 M5/M6 可用资金流特征：

- `feature_fund_flow_main_inflow_5d/10d/20d`
- `feature_fund_flow_super_inflow_10d`
- `feature_fund_flow_divergence_20d`
- `feature_fund_flow_main_inflow_streak`

股东特征当前作为可选增量家族：

- `feature_shareholder_holder_count_log`
- `feature_shareholder_holder_change_rate`
- `feature_shareholder_concentration_proxy`

M6 当前默认输入是 `price_volume + industry_breadth + fund_flow + fundamental`，暂不把 shareholder 作为主输入。

### 6.3 行业宽度

行业宽度由同一信号日行业内股票的表现聚合而来：

- 行业 20/60 日收益均值。
- 行业内 20 日正收益股票比例。
- 行业 20 日成交额均值。
- 行业低波代理。

它描述的是行业层面的共振，不是单票独立 alpha。

## 7. 截面处理

### 7.1 缩尾与 z-score

对同一交易日或信号月截面 `x(i,t)`，先按分位数缩尾：

```text
x_w(i,t) = min(max(x(i,t), Q_t(q_low)), Q_t(q_high))
```

再做截面标准化：

```text
z(i,t) = (x_w(i,t) - mean_t(x_w)) / (std_t(x_w) + eps)
```

月度链路还会生成缺失标记 `is_missing_<feature>`，并把原始特征和 z-score 特征一起纳入覆盖率诊断。

### 7.2 市值/行业中性化

研究模式下可对每个因子做横截面回归残差化：

```text
x(i,t) = alpha_t
       + beta_t * log_market_cap(i,t)
       + sum_g gamma(g,t) * 1[industry(i)=g]
       + epsilon(i,t)
```

残差 `epsilon(i,t)` 作为中性化后的因子。月度研究可对打分因子做信号日截面市值中性化，若配置了行业列则同时控制行业。

### 7.3 正交化

若因子相关性过高，可在训练或研究中启用正交化：

- Löwdin 对称正交化：对因子相关矩阵做特征分解，使用 `R^{-1/2}` 变换，最大程度保留原始信息且不引入列顺序偏好。
- Gram-Schmidt：按列顺序逐步对前序因子做残差化，适合因子有明确优先级时使用。

## 8. 截面打分模型

### 8.1 线性 composite

基础两因子模型：

```text
s(i,t) = w_m * z_momentum(i,t) + w_r * z_rsi(i,t)
```

权重非负并归一化。

### 8.2 多因子 composite_extended

扩展线性模型：

```text
s(i,t) = sum_f w_f * z_f(i,t)
```

`w_f` 按绝对值和归一化，允许负权重表达“越高越差”。若启用 IC 动态权重，同名因子会被滚动 ICIR 权重覆盖，未覆盖的因子回退静态配置。

### 8.3 XGBoost 截面排序

`sort_by=xgboost` 使用训练好的 `XGBRanker` 工件推理。训练时每个交易日或信号月是一个 query group：

```text
query group = {x(i,t), y(i,t) for i in U_t}
```

标签来自未来收益的组内等级或多 horizon rank fusion。目标是让未来相对更强的股票获得更高 `tree_score`，而不是精确回归收益。

`rank_ic_guard` 用验证集 Rank IC 的历史分布和最低阈值阻止退化模型发布。

### 8.4 深度序列模型

`deep_sequence` 使用最近 `seq_len` 的 OHLCV 序列，通过 LSTM / GRU / TCN / Transformer 输出 `deep_sequence_score`。它适合作为研究补充，当前不是默认主线。

## 9. 月度 M6 Learning-to-Rank

月度主模型入口是 `scripts/run_monthly_selection_ltr.py`。它消费 M2 canonical dataset，并复用 M4/M5 的 walk-forward 评价。

### 9.1 特征规格

默认 M6 特征族：

```text
price_volume + industry_breadth + fund_flow + fundamental
```

其中 price-volume baseline 包含：

```text
feature_ret_5d_z, feature_ret_20d_z, feature_ret_60d_z,
feature_realized_vol_20d_z, feature_amount_20d_log_z,
feature_turnover_20d_z, feature_price_position_250d_z,
feature_limit_move_hits_20d_z
```

### 9.2 分组排序标签

M6 将每个 `signal_date` 作为 query group。训练标签优先使用 `label_forward_1m_excess_vs_market`，并在每个月内转成 0 到 `relevance_grades-1` 的等级：

```text
relevance(i,m) = ceil(rank_pct(y_excess(i,m)) * grades) - 1
```

这样模型学习的是“同月截面内谁更靠前”，不是跨月份收益水平。

### 9.3 模型族

| 模型 | 说明 |
| --- | --- |
| `M6_xgboost_rank_ndcg` | XGBRanker，objective=`rank:ndcg`，eval_metric=`ndcg@20` |
| `M6_xgboost_rank_pairwise` | XGBRanker，objective=`rank:pairwise` |
| `M6_top20_calibrated` | XGBoost 二分类器预测未来 top20 bucket，再转截面分位 |
| `M6_ranker_top20_ensemble` | 固定 `0.60 * rank_ndcg + 0.40 * top20` |

所有 walk-forward 测试月只使用该月之前的历史月份训练。

### 9.4 当前研究结论

按 `docs/reports/2026-04/monthly_selection_m6_ltr_2026-04-29.md`：

- `U2_risk_sane + M6_xgboost_rank_ndcg` 通过 Top20 / Top30 watchlist gate。
- U1 口径未稳定优于 M5 对照。
- Top50 未通过 M5 对照。
- top20 classifier 和固定 ensemble 未提升为主模型。
- M6 仅进入 M7 研究报告和后续 regime-aware 复核，不进入生产。

M7 报告还暴露了当期推荐高度集中于非银金融的问题，这属于后续行业集中度和 regime-aware calibration 治理项。

## 10. Regime Switch

`src/market/regime.py` 用基准日收益序列判断市场状态。基准可以是 `510300`，也可以是 `market_ew_proxy`。

状态由短期累计收益、长期累计收益和短期年化波动率决定：

| 状态 | 判定倾向 |
| --- | --- |
| `bull` | 短期收益强、长期收益为正、波动未超阈值 |
| `bear` | 短期显著下跌，或短期/长期同时偏弱 |
| `oscillation` | 其余状态 |

线性权重调整分两层：

1. 硬状态乘数：牛市增强动量，熊市增强低波/大市值，震荡适度增强反转。
2. 可选连续动态权重：根据波动率超目标和趋势强度平滑调整风险暴露。

调整后按绝对值之和重新归一化，保持因子方向。

## 11. 组合优化

给定 Top-K 排序表，组合权重 `w` 满足长-only、非负、和约束。支持方法如下：

| 方法 | 目标 |
| --- | --- |
| `equal` | `w_i = 1/K` |
| `score` | 对非负分数归一化 |
| `tiered_equal_weight` | Top tier 和剩余股票分层等权 |
| `risk_parity` | 各资产风险贡献尽量相等 |
| `min_variance` | 最小化 `w' Sigma w` |
| `mean_variance` | 最大化 `mu'w - gamma * w'Sigma w` |

协方差估计：

- `ledoit_wolf`：样本协方差收缩，默认推荐。
- `sample`：原始样本协方差。
- `ewma`：指数加权协方差。
- `industry_factor`：行业因子 + 特异风险近似。

约束包括单票权重上限、行业权重上限、行业持仓只数上限、半 L1 换手约束和按市值分档的换手成本系数。

月度研究报告通常用等权评价 Top-K 排序质量；生产配置可根据 `portfolio.weight_method` 输出可执行权重。

## 12. 回测与评估

### 12.1 执行模式

回测引擎支持：

| 模式 | 含义 |
| --- | --- |
| `tplus1_open` | 使用 open-to-open 收益，贴近月度主口径 |
| `close_to_close` | 使用收盘到收盘收益，保留兼容 |
| `vwap` | close-to-close 收益上叠加 VWAP 执行冲击 |

`limit_up_mode` 处理新增或增持股票在开盘涨停无法买入的情况：

- `idle`：买入失败资金闲置。
- `redistribute`：失败权重按当日其他可买标的比例重分配。

### 12.2 成本

默认成本：

| 项 | bps |
| --- | --- |
| 买入佣金 | 2.5 |
| 卖出佣金 | 2.5 |
| 单边滑点 | 4.5 |
| 卖出印花税 | 5.0 |

双边约 19 bps。月度 M6/M7 的 `topk_excess_after_cost` 使用半 L1 换手乘以 `cost_bps` 的简化成本敏感性。

### 12.3 指标

常用指标包括：

- 累计收益、年化收益、年化波动、夏普、最大回撤。
- Top-K 平均收益、超额收益、命中率。
- Rank IC、Rank IC IR。
- Top-K minus next-K。
- quantile top-minus-bottom spread。
- 年份、市场状态、行业暴露和候选池宽度切片。
- 换手、成本拖累、涨停买入失败诊断。

## 13. IC 监控与因子治理

`src/features/ic_monitor.py` 提供：

- `ICRecord`：单次因子 IC 记录。
- `ICMonitor`：追加、查询和滚动统计历史 IC。
- `ICDecayAlert`：滚动 IC 均值低于阈值时触发衰减告警。
- `compute_and_persist_factor_ic`：计算并持久化因子 IC。

月度研究可通过 `signals.p1_factor_filter` 根据 IC 报告对因子做 keep / zero / flip / remove；也可通过 `signals.ic_weighting` 使用滚动 ICIR 动态权重。默认这些治理能力是可选项，不能替代正式回测和 promotion。

## 14. 研究治理与 promotion

仓库用三层机制避免研究结论污染生产：

1. 配置隔离：`config.yaml.backtest` 服务研究回测；`config.yaml.example` 和本地 `config.yaml` 服务月度运行。
2. 证据链：研究脚本输出 `research_topic`、`research_config_id`、`output_stem`、CSV/JSON/Markdown 和 manifest。
3. Promotion registry：只有 `configs/promoted/promoted_registry.json` 中的 `promoted_configs` 才能作为生产候选。

明确不是 promotion 终点的结果：

- `signal_diagnostic`
- `light_strategy_proxy`
- gray zone
- 单次 oracle
- 单次 M7 research report

新增 promoted 配置至少需要正式 full backtest、OOS、状态切片、边界诊断、执行口径、成本、universe、换手和人工确认全部可追溯。

## 15. 当前系统优先级

当前最重要的建模优先级是：

1. 保持 PIT-safe 数据和 open-to-open 标签一致，并维持“持有月最后一个交易日卖出、下一交易日买入”的生产时点。
2. 维护 U0/U1/U2 候选池定义，不把候选池写成手写 alpha 策略。
3. 继续用月度 M6/M7 验证 market-relative Top-K，而不是回到旧 replacement 目标。
4. 对 M7 暴露出的行业集中度做治理，不能把单月高集中推荐直接视为可交易策略。
5. 将任何进入生产的尝试都放进 promoted registry，而不是直接改默认配置。

## 16. 风险提示

模型分数只代表在当前数据、口径、候选池和历史样本下的相对排序。它可能受数据缺失、行业集中、风格漂移、市场状态变化、交易成本、开盘不可买、公告事件和模型过拟合影响。本项目输出不构成投资建议。
