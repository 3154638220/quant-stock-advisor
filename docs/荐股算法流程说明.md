# 荐股算法流程说明

本文档基于当前仓库实现，全面说明系统如何从行情数据出发，经因子构造、截面排序、组合优化，生成每日推荐股票池与组合权重，并阐明 LLM 新闻链路与 IC 监控模块在整体体系中的位置。

---

## 1. 总体目标

系统的核心任务不是预测某只股票的精确涨幅，而是在每个交易日收盘后，从全市场中稳定筛出一组相对更优、风险更可控的候选标的，最终形成可执行的组合权重。

完整主流程可概括为：

```text
行情数据 (AkShare → DuckDB)
    │
    ▼
股票池准备 & 数据质量检查
    │
    ▼
股票池预过滤（硬规则剔除妖股/死水/高位股）
    │
    ▼
因子计算（传统量价因子 + K 线结构高频降频代理因子）
    │
    ▼
截面标准化 → 中性化 → （可选）因子正交化
    │
    ▼
大盘状态分类 (Regime Switch)  ──→  动态调整因子权重
    │
    ▼
截面打分 / 模型排序
  ├── 线性组合 (composite / composite_extended)
  ├── XGBoost 排序模型（默认主线）
  └── 深度时序模型（LSTM / GRU / TCN / Transformer）
    │
    ▼
Top-K 推荐池候选
    │
    ▼
次日可买性过滤（停牌 / 一字涨停剔除）
    │
    ▼
组合权重生成（等权 / 按分数 / 风险平价 / 最小方差 / 均值-方差）
  + 单票上限 / 行业上限 / 换手约束 / Ledoit-Wolf 协方差收缩
    │
    ▼
推荐结果输出 (recommend_*.csv) & HTML 报告
    │
    ▼
事后评估 & 回测 (eval / run_backtest_eval)
```

---

## 2. 输入数据

主线荐股算法使用日线量价数据，主要字段：


| 字段         | 含义  |
| ---------- | --- |
| `open`     | 开盘价 |
| `high`     | 最高价 |
| `low`      | 最低价 |
| `close`    | 收盘价 |
| `volume`   | 成交量 |
| `turnover` | 换手率 |


**数据管道**：通过 `AkShare` 拉取（支持东财、新浪等多路径，由 `config.yaml` 的 `akshare.daily_source_preference` 控制，内置网络韧性与本地缓存），增量写入本地 `DuckDB`。结果输出目录由 `paths.results_dir` 控制，默认 `data/results`。

---

## 3. 执行时点与交易口径

系统默认在收盘后运行，用当日及以前的数据生成信号，不使用未来信息。

默认交易口径为：

1. `t` 日收盘后生成推荐。
2. 最早在 `t+1` 日开盘买入（A 股 T+1 约束）。
3. 前瞻收益标签采用 `t+1` 开盘到 `t+1+h` 开盘，比直接使用收盘到收盘更保守。

这也是系统强调「收盘后选股」和「次日可交易性」的原因。

---

## 4. 主流程分解

### 4.1 股票池准备

- 若命令行传入 `--symbols`，则只处理指定股票。
- 否则从 `DuckDB` 或 `universe_cache` 中读取全市场或缓存股票列表。
- 若未使用 `--skip-fetch`，会先执行一次日线增量更新（调用 `AkShare` 客户端，支持并发拉取与自动重试）。

### 4.2 数据质量检查

写入完成后，系统检查：

- OHLC 合法性（high ≥ close ≥ low, open 合理等）
- 行情缺失情况与空值比例告警
- 交易日跨度是否异常（`config.yaml` 的 `quality` 节控制告警阈值）

若存在问题会记录告警，不轻易中断全量跑批，避免单一数据源抖动影响整体流程。

### 4.3 股票池预过滤

在正式打分前，先做一层硬规则过滤（`src/market/tradability.py` 的 `prefilter_stock_pool`），排除：

- 近几日涨停或跌停次数过多的标的
- 换手率长期处于市场极低分位或极高分位的标的
- 当前价格已处于过去区间高位（`price_position` 接近 1）的标的

核心作用：先排除「妖股化」「死水股化」「绝对高位化」候选，降低后续模型误选概率。过滤参数统一在 `config.yaml` 的 `prefilter` 节中配置。

## 5. 因子计算

系统以宽表形式构造历史价格矩阵，再通过张量运算（GPU/CPU）批量计算因子，最终转成长表供排序和训练使用。

当前因子体系分为三大类：

### 5.1 传统量价因子

由 `src/features/tensor_base_factors.py` 实现：


| 因子                         | 说明                   |
| -------------------------- | -------------------- |
| `momentum`                 | N 日动量，衡量价格延续强度       |
| `rsi`                      | Wilder RSI，识别强弱与超买超卖 |
| `realized_vol`             | W 日实现波动率（滚动标准差）      |
| `atr`                      | Wilder 平均真实波幅，含跳空    |
| `turnover_roll_mean`       | W 日换手率均值，衡量活跃度       |
| `vol_to_turnover`          | 波动率/换手率，识别异常波动结构     |
| `vol_ret_corr`             | 量价相关系数（对数成交量 vs 收益率） |
| `short_reversal`           | 短期反转（短周期动量取负）        |
| `volume_skew_log`          | 对数成交量分布偏度            |
| `bias_short` / `bias_long` | 短/长周期乖离率，抑制追涨        |
| `recent_return`            | 近端涨幅惩罚因子，抑制过热        |
| `max_single_day_drop`      | 窗口内最大单日跌幅，识别尾部风险     |
| `price_position`           | 价格在区间内的百分位           |
| `log_market_cap`           | 对数市值代理，控制小盘偏好        |


### 5.2 K 线结构高频降频代理因子

由 `src/features/intraday_proxy_factors.py` 实现，在无分钟数据的情况下，通过日线 OHLCV 内部结构近似刻画日内微观行为：


| 因子                    | 说明                |
| --------------------- | ----------------- |
| `intraday_range`      | 日内振幅/开盘价，妖股过滤     |
| `upper_shadow_ratio`  | 上影线比率，上方抛压        |
| `lower_shadow_ratio`  | 下影线比率，下方支撑        |
| `close_open_return`   | 日内涨跌幅（收/开-1），尾盘强弱 |
| `overnight_gap`       | 隔夜跳空（开/前收-1），消息冲击 |
| `tail_strength`       | 尾盘强度滚动均值，持续买力     |
| `volume_price_trend`  | 量价趋势因子（VPT），量价同向性 |
| `intraday_range_skew` | 振幅偏度，识别偶发性大幅震荡    |


### 5.3 因子组合入口

`compute_base_factor_bundle` 整合传统量价因子与 K 线结构代理因子，统一输出供后续流程使用。

## 6. 截面标准化与中性化

原始因子量纲各异、极值明显，不可直接横向比较，系统采用「同日截面处理」的方式统一尺度。

1. **Winsorize 去极值**：按分位数缩尾，消除极端异常值影响。
2. **截面 Z-score**：同日标准化，使各因子变为相对当日市场均值的无量纲指标。
3. **缺失值填充**：Z-score 后以当日截面中位数填充缺失值。
4. **中性化**（`src/features/neutralize.py`）：
  - 全市场去均值（截面去中心化）
  - 行业内去均值（控制行业共性暴露）
  - 市值暴露控制：`log_market_cap` 已显式纳入特征，模型可直接学习市值风格惩罚；研究模式下可进一步做行业+市值双重回归残差化。
5. **因子正交化**（`src/features/orthogonalize.py`，可选，由 `config.yaml` 的 `orthogonalize.enabled` 控制）：
  - **对称正交化（Löwdin）**：在所有使因子两两不相关的线性变换中，最大程度保留原始信息内容，各因子地位对称。
  - **Gram-Schmidt 正交化**：按列优先级逐步去冗余，适合因子有明确优先级的场景。

---

## 7. 因子 IC 监控

`src/features/ic_monitor.py` 提供持续的因子有效性监控能力：

- `**ICMonitor`**：按日追加 IC 记录（`ICRecord`），支持历史 IC 序列查询。
- `**ICDecayAlert**`：滚动窗口内 IC 均值低于告警阈值时触发衰减告警，用于识别因子失效迹象。
- `**compute_and_persist_factor_ic**`：计算因子 IC 并持久化，为模型诊断与策略维护提供数据支撑。

---

## 8. 大盘状态切换（Regime Switch）

`src/market/regime.py` 在生成组合打分之前，先对当前大盘状态进行分类，并根据状态动态调整因子权重（仅对 `composite` / `composite_extended` 排序键生效）。

**状态判断**：基于基准标的（默认 510300 沪深 300 ETF）的短期累积收益（20 日）、长期累积收益（60 日）和年化波动率，判断为牛市/熊市/震荡市。

**权重调整**：


| 市场状态 | 增强                  | 削弱       |
| ---- | ------------------- | -------- |
| 牛市   | 动量类 ×1.5            | 反转类 ×0.3 |
| 熊市   | 低波/大市值 ×2.0，市值 ×1.5 | 动量类 ×0.4 |
| 震荡   | 反转类 ×1.3            | —        |


调整后按绝对值之和重新归一化，正负方向不变。每日推荐 CSV 中的 `regime` 列记录当日判断结果。

## 9. 打分与排序

系统支持多种排序方式，由 `config.yaml` 的 `signals.sort_by` 控制（`src/models/rank_score.py`）。

### 9.1 线性组合模式

- `**composite`**：动量 + RSI 加权组合。
- `**composite_extended**`：引入全部量价因子、K 线结构代理因子、风险控制因子，支持负权重表达「越高越差」的偏好，权重按绝对值之和归一化。

### 9.2 机器学习排序模式

- `**xgboost**`（当前默认主线）：使用预训练的 `XGBRanker` 工件，对最新截面因子（共 23 个，含全部传统量价+K 线结构代理+风险控制因子）进行 pairwise 排序打分。训练时可通过 `--orthogonalize` 启用 Löwdin 正交化。
- `**deep_sequence**`（可选扩展）：基于 OHLCV 序列的深度学习模型（LSTM / GRU / TCN / Transformer），适合作为研究补充，不作为当前默认主线。

**XGBoost 训练核心机制**：

- 每个交易日截面构成一个 query group，标签为前瞻收益转化的稠密整数等级。
- 目标函数为 `rank:pairwise`，最大化截面内相对顺序正确率。
- 验证指标为 RankIC（Spearman 秩相关），越高说明模型越擅长把未来更强的股票排到前面。
- 支持 `rank_ic_guard`（最低 RankIC 阈值校验）避免使用退化模型。

---

## 10. 生成推荐池

排序完成后：

1. 按分数从高到低排序。
2. 先取一个放大的候选池（比最终 Top-K 更多）。
3. 做「次日可买性过滤」（`filter_recommend_tradable_next_day`）：
  - 剔除停牌标的
  - 剔除次日开盘预计一字涨停等难以成交的标的
4. 最终截取 `top_k` 作为推荐结果（`config.yaml` 的 `signals.top_k` 配置）。

最终输出不只是「分数高」，还要求「次日更可能买得到」。

---

## 11. 组合权重生成

推荐池生成后，系统按配置生成稳健的组合权重（`src/portfolio/`）。

### 11.1 支持的权重方法


| 方法              | 说明                  |
| --------------- | ------------------- |
| `equal`         | 等权                  |
| `score`         | 按分数归一化（分数非负部分）      |
| `risk_parity`   | 风险平价（ERC），各资产风险贡献均衡 |
| `min_variance`  | 最小方差                |
| `mean_variance` | 均值-方差（含风险厌恶系数 γ）    |


### 11.2 协方差估计

使用最近 M 日日收益，支持以下协方差估计方法（`src/portfolio/covariance.py`）：

- `**ledoit_wolf`**（默认）：Ledoit-Wolf 收缩协方差，缓解样本协方差噪声大与矩阵病态问题。
- `**ewma**`：指数加权移动协方差，对近期数据赋予更高权重（半衰期可配置）。
- `**industry_factor**`：行业因子模型协方差，先分解行业共同因子，再估计残差协方差，适合行业暴露较高时使用。
- `**sample**`：原始样本协方差（不推荐直接使用）。

估计后叠加 ridge 稳定化项 `λI`，最终协方差 = 收缩估计 + λI。

### 11.3 组合约束

- **单票权重上限**：超限部分迭代截顶后重分配给未触顶股票。
- **行业权重上限**：防止对单一行业过度暴露。
- **换手约束**：半边换手超限时，新旧权重做凸组合收缩（η 由换手上限决定）。
- **换手成本模型**：支持按市值分档的换手拖累系数（`portfolio.turnover_cost_model`），大盘股和小盘股的冲击成本有所不同。

这一步把「选股结果」转化为「可执行组合」，并控制单一高波动股票对净值的伤害。

## 12. 输出结果

主线荐股流程最终输出 `recommend_*.csv`，包含：


| 列     | 说明                              |
| ----- | ------------------------------- |
| 股票代码  | symbol                          |
| 各因子值  | 原始与标准化值                         |
| 排序分数  | `score`                         |
| 推荐理由  | 中文化摘要（`build_recommend_reason`） |
| 权重    | `weight`                        |
| 排序方式  | `sort_by`                       |
| 市场状态  | `regime`                        |
| 截止交易日 | `asof_trade_date`               |


文件默认写入 `data/results/`。同时可生成本地 HTML 推荐解释报告（`scripts/recommend_report.py`），用于快速检查「模型为什么把某只股票排到前面」。

## 13. 事后评估与回测

### 13.1 推荐结果事后评估

`scripts/daily_run.py eval` 调用 `src/cli/eval_recommend.py`，对历史推荐 CSV 做前向收益标注与组合表现分析：

- 单票未来收益（T+1 开盘买入口径）
- Top-K 平均前瞻收益与命中率
- 含/不含交易成本的组合收益对比

### 13.2 全组合向量回测

`scripts/run_backtest_eval.py` 调用 `src/backtest/engine.py` 的 `run_backtest`，支持更完整的回测：

- 默认使用「前一日权重 × 当日收益」，执行模式可选 `close_to_close` 或 `tplus1_open`。
- 涨停买入失败处理：支持「资金闲置」和「权重顺延重分配」（默认）两种模式。
- 交易成本模型：佣金买卖各 2.5 bps，滑点 4.5 bps/侧，印花税（卖侧）5 bps，合计双边约 **19 bps**。
- 常见绩效指标：累计/年化收益、年化波动率、夏普比率、最大回撤、换手率。

### 13.3 Walk-forward 时间切片验证

`src/backtest/walk_forward.py` 支持滚动 Walk-forward 验证：每一折仅用训练窗拟合，在测试窗输出组合表现，多折聚合为稳定性指标，比一次性切分更接近真实部署环境。

## 14. LLM 新闻辅助链路

仓库中还有一条独立的 LLM 链路（`src/llm/`），当前使用本地 Ollama 服务：

```text
东方财富飙升榜 / 人气榜
    │
    ▼
拉取个股相关新闻（AkShare + 网络韧性缓存）
    │
    ▼
LLM 判断是否为真实新闻驱动（OllamaClient + prompts）
  ├── 关注度过滤 (AttentionScanner)
  ├── 新闻情绪分析 (NewsAnalyzer)
  └── 财务摘要分析 (FinancialAnalyzer)
    │
    ▼
输出新闻驱动股票清单 (JSON / CSV)
    │
    ▼
生成 Markdown / HTML 报告 (llm_analysis_report.py)
```

**用途**：

- 从「关注度飙升」股票中筛出真正由新闻催化驱动的标的
- 区分新闻事件与纯市场情绪波动
- 为人工复盘或事件驱动研究提供补充信息

**定位**：目前是独立输出的辅助分析模块，尚未直接并入主线量化荐股打分体系。LLM 配置（模型、Ollama URL、超时/重试、扫描规模等）统一在 `config.yaml` 的 `llm` 节中配置。

## 15. 系统核心优先级总结

从「当前最稳妥、最贴近实盘」的角度，各模块优先级大致为：

1. **股票池预过滤**：基础门槛筛选，排除不可取标的
2. **传统量价因子 + K 线结构高频降频代理因子**：23 因子联合构造
3. **截面标准化、行业去均值、市值暴露控制**；可选因子正交化（Löwdin）
4. **大盘状态分类（Regime Switch）**：动态适配市场环境
5. **XGBoost 截面排序模型**：核心打分主线
6. **次日可买性过滤**：保证实际可执行
7. **Ledoit-Wolf / EWMA 稳健协方差 + 风险平价/最小方差优化**：组合风险控制
8. **单票上限 + 行业上限 + 换手约束**：实盘可行性约束
9. **涨停重分配 + 保守双边成本（~19 bps）回测评估**：接近真实绩效
10. **IC 监控 + eval 复盘 + walk-forward 稳定性检验**：持续质量保障
11. **LLM 新闻链路**：辅助分析，非主线

## 16. 常用命令

### 主线荐股

```bash
# 增量拉取 + 生成推荐（最多处理 100 只股票）
python scripts/daily_run.py --max-symbols 100

# 跳过拉取，直接推荐 Top 20
python scripts/daily_run.py run --skip-fetch --top-k 20

# 只处理指定股票
python scripts/daily_run.py --symbols 600519,000001

# 使用 XGBoost 模型
python scripts/daily_run.py run --sort-by xgboost
```

### 推荐结果评估

```bash
# 对最新推荐结果做 5 日前向收益评估
python scripts/daily_run.py eval --latest --horizon 5

# 对指定推荐文件评估
python scripts/daily_run.py eval --csv data/results/recommend_2026-03-27.csv
```

### 全组合回测

```bash
# 对历史推荐结果做向量回测（含成本、walk-forward）
python scripts/run_backtest_eval.py
```

### 模型训练

```bash
# 训练 XGBoost 排序模型
python scripts/train/train_xgboost.py

# 训练深度时序模型
python scripts/train/train_deep_sequence.py --arch gru --seq-len 30
```

### 推荐 HTML 报告

```bash
# 生成最新推荐的 HTML 解释报告
```
