# 荐股算法流程说明

更新时间：`2026-04-30`

本文档说明当前仓库如何从行情、基本面、资金流等数据生成股票推荐名单。需要先区分两个层级：

| 层级 | 入口 | 目标 | 当前状态 |
| --- | --- | --- | --- |
| 日更推荐 | `scripts/daily_run.py` | 收盘后用最新截面因子生成 `recommend_*.csv` | 本地运行入口，默认 `composite_extended`，可切换 XGBoost / deep sequence |
| 月度选股研究 | `scripts/run_monthly_selection_*.py` | 月末从可交易股票池中选出未来一个月更可能跑赢市场的 Top-K | 当前 canonical 研究主线，M7 已输出研究版推荐报告，尚未 promoted |

截至仓库当前 promoted registry 记录，`configs/promoted/promoted_registry.json` 中 `promoted_configs` 为空。也就是说，月度 M7 推荐名单和历史 P1/R2/R2B 研究结果都不是生产交易指令。

---

## 1. 总体架构

```text
AkShare / 本地缓存
    │
    ▼
DuckDB: 日线、基本面、资金流、股东、行业映射
    │
    ├───────────────────────────────────────────────┐
    ▼                                               ▼
日更推荐链路                                      月度选股研究链路
daily_run.py                                     run_monthly_selection_*.py
    │                                               │
    ▼                                               ▼
日线张量因子 + PIT 基本面 + 可选 LLM 因子          M2 canonical monthly dataset
    │                                               │
    ▼                                               ▼
预过滤 / 标准化 / 中性化 / Regime / IC 动态权重     U0/U1/U2 候选池 + open-to-open 标签
    │                                               │
    ▼                                               ▼
composite_extended / XGBoost / deep_sequence       Oracle / baseline / multisource / LTR
    │                                               │
    ▼                                               ▼
次日可买性过滤 + 组合权重                          M7 研究版 Top-K 推荐报告
    │                                               │
    ▼                                               ▼
recommend_*.csv / HTML 报告 / eval                 monthly_selection_* 证据链
```

两条链路共享数据、特征处理、可交易性和回测思想，但用途不同。日更链路偏“今天收盘后给出一个本地推荐 CSV”；月度链路偏“严格研究口径下验证每月 Top-K 排序模型是否有稳定超额”。

## 2. 数据输入

### 2.1 行情数据

主表默认是 `a_share_daily`，字段包括：

| 字段 | 含义 |
| --- | --- |
| `symbol` | 6 位股票代码 |
| `trade_date` | 交易日 |
| `open` / `high` / `low` / `close` | OHLC |
| `volume` | 成交量 |
| `amount` | 成交额，月度链路用于流动性和市值代理 |
| `turnover` | 换手率 |
| `pct_chg` | 涨跌幅，部分诊断使用 |

行情由 `src/data_fetcher/` 拉取并写入 DuckDB。AkShare 源、复权、超时、重试、并发和缓存回退由 `config.yaml` 的 `akshare` 节控制。

### 2.2 扩展数据

| 数据 | 入口 | 用途 |
| --- | --- | --- |
| 行业映射 | `scripts/build_industry_map.py` | 行业中性化、行业暴露、月度行业宽度特征 |
| 基本面 | `scripts/fetch_fundamental.py` / `FundamentalClient` | PIT 估值、质量、成长、现金流因子 |
| 资金流 | `scripts/fetch_fund_flow.py` | 主力净流入、超大单、资金流背离、连续流入 |
| 股东数据 | `scripts/fetch_shareholder.py` | 股东户数变化、筹码集中度代理 |
| LLM 新闻关注度 | `scripts/llm_daily_analysis.py` | 独立事件驱动扫描；日更链路可读取当日 CSV 构造 `llm_sentiment_z` |

月度研究链路会显式输出覆盖率、缺失标记和特征可用时间范围；低覆盖特征不会直接被当作稳定 alpha。

## 3. 交易时点与标签口径

系统默认在收盘后运行，不使用未来数据。

日更推荐口径：

1. `t` 日收盘后生成信号。
2. 最早在 `t+1` 日开盘买入。
3. 事后评估默认使用 `t+1` 开盘买入到 `t+1+h` 开盘卖出的收益。
4. `daily_run.py` 只生成推荐和权重，不维护真实 T+1 仓位状态机。

月度研究口径：

1. 信号日是每月最后一个交易日。
2. 候选股票必须在下一个交易日开盘可买。
3. 标签为从信号日后第一个 open-to-open 收益开始，复利持有到下一月换仓前后的未来一个月收益。
4. 主评价使用市场等权 open-to-open 超额、行业中性超额、Top-K 与 next-K 边界、Rank IC、状态切片和换手成本敏感性。

## 4. 日更推荐链路

日更主入口是 `scripts/daily_run.py`。

### 4.1 股票池准备

优先级如下：

1. 命令行 `--symbols` 指定股票。
2. 否则使用 `list_default_universe_symbols()` 从 AkShare、DuckDB 和本地 universe cache 组合股票池。
3. `--max-symbols` 只用于调试截断。
4. 未传 `--skip-fetch` 时先做日线增量写入。

写入后会调用 `quality_report()` 检查 OHLC 合法性、空值、异常 gap 等。质量告警通常记录日志，不轻易中断全量跑批。

### 4.2 硬规则预过滤

`prefilter_stock_pool()` 在打分前排除明显不可取或高风险样本：

| 规则 | 说明 |
| --- | --- |
| ST / `*ST` | 若日线中有名称列，则剔除包含 ST 的股票 |
| 短期涨跌停路径 | 近 `limit_move_lookback` 日涨跌停次数超过阈值 |
| 极端换手 | 当日换手处于截面极低或极高分位 |
| 绝对高位 | 当前价格处于过去 `price_position_lookback` 区间高位 |

这些规则只做风险准入，不应被解释为 alpha。

### 4.3 因子计算

日更链路将日线长表透视成宽表，再用 PyTorch 张量批量计算因子。当前输出包括：

| 因子族 | 因子 |
| --- | --- |
| 趋势/反转 | `momentum`、`short_reversal`、`recent_return` |
| 强弱/波动 | `rsi`、`atr`、`realized_vol`、`max_single_day_drop` |
| 量价结构 | `turnover_roll_mean`、`vol_ret_corr`、`vol_to_turnover`、`volume_skew_log` |
| 价格位置 | `bias_short`、`bias_long`、`price_position`、`log_market_cap` |
| 周线结构 | `weekly_kdj_k/d/j`、`weekly_kdj_oversold`、`weekly_kdj_oversold_depth`、`weekly_kdj_rebound` |
| K 线代理 | `intraday_range`、`upper_shadow_ratio`、`lower_shadow_ratio`、`close_open_return`、`overnight_gap`、`tail_strength`、`volume_price_trend`、`intraday_range_skew` |
| PIT 基本面 | `pe_ttm`、`pb`、`ev_ebitda`、`roe_ttm`、`net_profit_yoy`、`gross_margin_change`、`debt_to_assets_change`、`ocf_to_net_profit` 等 |
| 可选事件因子 | `llm_sentiment_z`，来自当日 `llm_attention_*.csv` |

基本面通过 `FundamentalClient.load_point_in_time()` 按 `asof_date` 读取，避免使用未来公告数据。若基本面表不可用，日更流程会回退到技术面因子。

### 4.4 标准化与中性化

日更单截面上会执行：

1. 因子数值清洗，替换无穷值。
2. 对基本面因子做缩尾、缺失填充和可选市值/行业中性化。
3. 对打分因子做市值回归残差中性化，若配置了行业列则同时控制行业虚拟变量。
4. 线性打分阶段对参与列做截面 z-score。

若 `orthogonalize.enabled=true`，训练或研究流程还可启用 Löwdin 对称正交化或 Gram-Schmidt 正交化，减少因子冗余。

### 4.5 打分排序

`signals.sort_by` 控制日更排序方式：

| `sort_by` | 说明 |
| --- | --- |
| `momentum` | 直接按动量排序 |
| `rsi` | 直接按 RSI 排序 |
| `composite` | 动量 + RSI 线性组合 |
| `composite_extended` | 默认日更主线，多因子 z-score 线性组合，支持负权重 |
| `xgboost` | 使用 `signals.tree_model.bundle_dir` 中的 XGBoost 工件推理 |
| `deep_sequence` | 使用 OHLCV 序列模型工件推理 |

`composite_extended` 的权重来自 `config.yaml`，当前包含技术面、K 线结构、周线 KDJ、基本面、资金流和 LLM 情绪等可用列。覆盖率低于 20% 的稀疏因子会在单次日更中临时跳过，避免全量 `dropna`。

### 4.6 Regime Switch 与 IC 动态权重

当 `regime.enabled=true` 且排序键为 `composite` / `composite_extended` 时，系统会用基准日收益判断市场状态：

| 状态 | 倾向 |
| --- | --- |
| `bull` | 增强动量，压低反转 |
| `bear` | 增强低波、防御和大市值，压低动量 |
| `oscillation` | 保持均衡，适度提升反转 |

若 `signals.ic_weighting.enabled=true`，还可读取 `data/cache/ic_weights.json` 或 `data/logs/ic_monitor.json`，用滚动 ICIR 覆盖静态因子权重。

### 4.7 次日可买性过滤

排序后先取放大的候选池，再调用 `filter_recommend_tradable_next_day()`：

- 次日无行情时不剔除，只记录日志。
- 若次日开盘停牌近似、OHLC 无效或成交量为 0，则剔除。
- 若次日开盘触及对应板块涨停价，视为一字涨停不可买，剔除。

最终截取 `top_k` 输出，确保推荐名单不仅“分高”，也尽量“买得到”。

### 4.8 组合权重

`src/portfolio/weights.py` 将推荐名单转成组合权重。支持：

| 方法 | 说明 |
| --- | --- |
| `equal` | 等权 |
| `score` | 分数非负部分归一化 |
| `tiered_equal_weight` | Top tier 与 bottom tier 分层等权 |
| `risk_parity` | 风险平价 |
| `min_variance` | 最小方差 |
| `mean_variance` | 均值-方差 |

协方差估计支持 `ledoit_wolf`、`sample`、`ewma`、`industry_factor`，并叠加 ridge 稳定化。组合约束包括单票上限、行业权重上限、半 L1 换手约束和按市值分档的换手成本系数。

### 4.9 日更输出

日更推荐 CSV 默认写入 `data/results/recommend_<date>.csv`，常见列包括：

| 列 | 说明 |
| --- | --- |
| `rank` | 推荐排名 |
| `symbol` | 股票代码 |
| `asof_trade_date` | 信号截止交易日 |
| 因子列与 `z_*` 列 | 原始因子与截面标准化结果 |
| `composite_extended_score` / `tree_score` / `deep_sequence_score` | 对应排序分数 |
| `regime` | 当日市场状态 |
| `weight` | 组合权重 |
| `sort_by` | 排序方式 |
| `recommend_reason` | 中文推荐理由摘要 |

## 5. 月度选股研究链路

月度链路是当前主研究方向，目标是每月输出可解释、PIT-safe、可回测的 Top-K 研究推荐名单。

### 5.1 M2 canonical dataset

入口：

```bash
python scripts/run_monthly_selection_dataset.py --config config.yaml.backtest
```

核心产物是 `data/cache/monthly_selection_features.parquet`。它包含：

- 每月最后一个交易日的截面特征。
- 下一个交易日开盘可买性。
- `U0_all_tradable`、`U1_liquid_tradable`、`U2_risk_sane` 三层候选池。
- open-to-open 月度标签、市场超额、行业中性超额、未来 top bucket 标签。
- 候选池宽度、拒绝原因、特征覆盖率和标签分布报告。

### 5.2 候选池版本

| pool | 定义 | 用途 |
| --- | --- | --- |
| `U0_all_tradable` | 当前 OHLCV 有效且次日开盘可买 | 判断全可交易空间 |
| `U1_liquid_tradable` | U0 + 历史长度 + 20 日成交额门槛 | 默认训练候选池 |
| `U2_risk_sane` | U1 + 排除极端涨跌停路径、极端波动/换手、绝对高位 | 当前 M7 研究报告 watchlist 口径 |

候选池只做可交易和极端风险过滤，不负责判断股票是否有 alpha。

### 5.3 M3 到 M7

| 阶段 | 入口 | 说明 |
| --- | --- | --- |
| M3 Oracle | `run_monthly_selection_oracle.py` | 评估候选空间中事后 Top-K / top bucket 上限 |
| M4 Baseline | `run_monthly_selection_baselines.py` | price-volume 单因子、线性组合和基础 ML baseline |
| M5 Multisource | `run_monthly_selection_multisource.py` | 增量评估行业宽度、资金流、基本面、股东特征 |
| M6 LTR | `run_monthly_selection_ltr.py` | 按 `signal_date` query group 训练 XGBoost ranker / top20 classifier |
| M7 Report | `run_monthly_selection_report.py` | 用 M6 历史证据和最新信号月生成研究版 Top-K 推荐报告 |

M6 当前 watchlist 口径是 `U2_risk_sane + M6_xgboost_rank_ndcg + Top20/Top30`。M7 会输出推荐名单、特征贡献、行业暴露、风险摘要、历史 leaderboard、Rank IC、分桶和切片附件。它不写入 promoted registry，也不生成交易指令。

### 5.4 月度报告的重点诊断

M7 报告不仅看推荐分数，还会披露：

- 当月推荐名单和 `buyability`。
- 历史 M6 leaderboard、Top-K 超额、成本后超额。
- Rank IC、quantile spread、Top-K vs next-K。
- 年份、市场状态、行业暴露切片。
- 候选池宽度和拒绝原因。
- 特征覆盖率、特征重要性和单票特征贡献。
- 风险摘要和上月持仓重合。

如果当月推荐高度集中于单一行业，报告会暴露该问题；这属于后续 regime-aware calibration 与集中度治理要解决的风险，不等价于生产就绪。

## 6. 回测与事后评估

日更推荐评估：

```bash
python scripts/daily_run.py eval --latest --horizon 5
python scripts/daily_run.py eval --csv data/results/recommend_2026-03-27.csv
```

正式回测：

```bash
python scripts/run_backtest_eval.py --config config.yaml.backtest
```

回测支持：

- `tplus1_open`、`close_to_close`、`vwap`。
- 涨停买入失败 `idle` 或 `redistribute`。
- 双边约 19 bps 的默认保守成本。
- 月频/周频等调仓规则。
- walk-forward、时间切片、状态切片、行业暴露和换手诊断。

## 7. 常用命令

```bash
# 日更推荐
python scripts/daily_run.py --max-symbols 100
python scripts/daily_run.py run --skip-fetch --top-k 20
python scripts/daily_run.py run --sort-by xgboost

# 推荐解释与事后评估
python scripts/recommend_report.py --latest
python scripts/daily_run.py eval --latest --horizon 5

python scripts/run_monthly_selection_dataset.py --config config.yaml.backtest

python scripts/run_monthly_selection_baselines.py --config config.yaml.backtest

python scripts/run_monthly_selection_ltr.py --config config.yaml.backtest
python scripts/run_monthly_selection_report.py --config config.yaml.backtest

# 回测
python scripts/run_backtest_eval.py --config config.yaml.backtest

# LLM 新闻辅助分析
python scripts/llm_daily_analysis.py --with-macro
python scripts/llm_analysis_report.py --latest
```

## 8. 风险提示

系统输出的是研究筛选结果，不构成投资建议。实际交易还需要独立处理公告、流动性、盘中成交、行业事件、仓位上限、止损和个人风险约束。未进入 `configs/promoted/promoted_registry.json` 的任何研究结果，都不能视为生产策略。
